🎯 Objetivo: Identificar el tipo, topología y elementos de la red del aula o de casa.

🛠️ Material necesario:

  • PC conectado a la red del aula o doméstica

🎯 Objetivo: Leer la configuración de red del equipo e identificar IP privada, máscara, gateway y DNS.

🛠️ Material necesario:

  • PC con Windows y Ubuntu (VirtualBox)
  • Acceso a terminal/cmd

📋 Procedimiento:

  1. En Windows, abre cmd y ejecuta ipconfig /all. Identifica y anota cada campo.
  2. Determina si tu IP es privada o pública. Indica en qué rango (10.x.x.x, 172.16-31.x.x, o 192.168.x.x) está.
  3. Calcula cuántas IPs puede tener tu red con esa máscara de subred. Pista: 255.255.255.0 → 254 equipos.
  4. En Ubuntu, ejecuta resolvectl status y anota los servidores DNS configurados.
  5. Comprueba si tus DNS funcionan: nslookup google.com (Windows) o dig google.com (Ubuntu).
  6. Responde: ¿Qué pasaría si cambiaras la puerta de enlace a una IP incorrecta? Pruébalo y anota qué ocurre.

🎯 Objetivo: Utilizar las herramientas de diagnóstico para verificar la conectividad paso a paso.

🛠️ Material necesario:

  • VMs Windows y Ubuntu con red configurada
  • Acceso a Internet en las VMs (modo NAT)

📋 Procedimiento:

  1. En Windows: ipconfig — anota la IP y el gateway.
  2. Ping a tu propia IP: ping 127.0.0.1. ¿Qué verifica esto?
  3. Ping a la puerta de enlace. Si no responde, ¿qué parte de la red falla?
  4. Ping a 8.8.8.8. Si falla: ¿hay problema de routing o de DNS?
  5. Ping a google.com. Si este falla pero 8.8.8.8 funciona: ¿cuál es el problema?
  6. Ejecuta tracert google.com (Windows) o traceroute google.com (Ubuntu). Anota cuántos saltos hay.
  7. Repite los pasos 1-6 en Ubuntu usando ip addr, ping y traceroute.
  8. Elabora una tabla comparativa Windows/Ubuntu con los resultados.

📋 Procedimiento:
