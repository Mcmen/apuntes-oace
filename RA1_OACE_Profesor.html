B1

Solucionario · ¿Qué es un sistema operativo? Funciones, tipos y ejemplos

⏱️ 110 min
✅ Solución · Ejercicio 1.1 · Investigación de SO

1. Cajero automático: La mayoría usa Windows Embedded / Windows IoT (versiones recortadas de Windows para dispositivos especializados). Algunos usan Linux. En España, muchos cajeros de grandes bancos migraron de Windows XP Embedded a Windows 10 IoT.

2. PlayStation 5: Usa Orbis OS, un sistema operativo basado en FreeBSD (una variante de Unix de código abierto). No usa Linux directamente, aunque comparte raíces Unix.

3. Tesla Model 3: La pantalla central ejecuta un Linux personalizado por Tesla, con una interfaz gráfica construida en Qt. El sistema de conducción autónoma usa un sistema operativo de tiempo real aparte.

4. Router WiFi doméstico: La mayoría usan Linux embebido (OpenWrt, DD-WRT, Tomato, o firmware propietario del fabricante basado en Linux).

5. Apple Watch: Usa watchOS, un sistema operativo basado en iOS y Darwin (Unix), desarrollado por Apple exclusivamente para el reloj.

✅ Solución · Refuerzo 1.1 · Relaciona funciones del SO

1. Gestión del procesador → Cuando tienes varias pestañas del navegador abiertas y música de fondo, el SO reparte el tiempo de CPU entre todas.
2. Gestión de la memoria → Al abrir LibreOffice Writer, el SO le asigna un espacio en la RAM, separado del que usa Firefox.
3. Gestión del almacenamiento → Cuando guardas un archivo, el SO decide en qué sector del disco se escribe físicamente.
4. Gestión de dispositivos → Al conectar un pendrive, el SO carga el driver correspondiente y lo hace accesible.
5. Gestión de usuarios y seguridad → Cuando intentas instalar un programa, el SO te pide la contraseña del administrador.
6. Interfaz de usuario → El escritorio con iconos y la barra de tareas que ves al encender el ordenador.

✅ Solución · Profundización 1.1 · RTOS

Un RTOS (Real-Time Operating System) es un sistema operativo diseñado para garantizar que ciertas tareas se ejecutan en un plazo de tiempo máximo predecible. A diferencia de Windows o Linux (tiempo compartido), un RTOS no prioriza la "equidad" entre procesos sino el cumplimiento estricto de plazos.

Tres ejemplos de dispositivos con RTOS:
1. Marca-pasos cardíaco: Si el SO no responde a tiempo, el impulso eléctrico podría llegar en el momento equivocado del ciclo cardíaco, causando una arritmia potencialmente fatal.
2. Sistema de frenos ABS de un coche: Si el SO tarda más de unos milisegundos en responder al detectar el bloqueo de una rueda, el coche podría derrapar y perder el control.
3. Control de vuelo fly-by-wire de un avión (Airbus A320): Las órdenes del piloto deben procesarse en tiempo real. Un retraso en la respuesta podría causar oscilaciones peligrosas o pérdida de control.

¿Windows o Ubuntu para un avión? No. Ambos son sistemas de tiempo compartido y no pueden garantizar tiempos de respuesta. Un sistema de control de vuelo crítico usa RTOS certificados como VxWorks, Integrity-178B o QNX, que están diseñados y certificados específicamente para entornos donde un fallo puede costar vidas humanas.

✅ Solución · Test rápido del alumno · B1

1. Pregunta: ¿Cuál es la función principal de un sistema operativo?

Respuesta correcta: A) Gestionar los recursos del ordenador y hacer de intermediario entre el hardware y las aplicaciones.

💡 El SO no es solo el escritorio (opción B) ni solo el antivirus (opción C). Su función es coordinar TODO: procesador, memoria, disco y aplicaciones.

2. Pregunta: ¿Qué pasaría si no tuvieras sistema operativo?

Respuesta correcta: A) El ordenador encendería pero no podrías hacer nada útil con él.

💡 El hardware se encendería (la CPU ejecutaría el POST), pero sin SO no habría interfaz, no podrías abrir programas ni gestionar archivos.

✅ Solución · Test rápido del alumno · B2

Pregunta: ¿Qué tipo de SO necesitas para controlar los frenos de un coche?

Respuesta correcta: SO de tiempo real (RTOS).

💡 Un portátil de clase usa SO de tiempo compartido (multitarea). Un móvil es multiusuario (con perfiles) y multitarea. Solo el control de frenos requiere respuesta garantizada en microsegundos → RTOS.

✅ Solución · Test rápido del alumno · B3

Pregunta: ¿Qué capa se encarga de detectar un USB que acabas de conectar?

Respuesta correcta: Kernel + Drivers.

💡 El fondo de escritorio lo gestiona el Shell (GUI), no el kernel directamente. Ejecutar LibreOffice implica al kernel (lanzar proceso) pero la 'ejecución' visible es una responsabilidad del Shell. Detectar hardware nuevo es siempre Kernel + Driver.

✅ Solución · Test rápido del alumno · B4

Pregunta: ¿Qué es el POST?

Respuesta correcta: Un chequeo rápido que hace el ordenador al encenderse.

💡 POST = Power On Self Test. No es el programa que abre el escritorio (eso es el bootloader + kernel) ni un virus. Es el firmware (BIOS/UEFI) comprobando que CPU, RAM y periféricos básicos funcionan antes de buscar el SO.

✅ Solución · Test rápido del alumno · B5

Pregunta: ¿Por qué usamos VirtualBox en lugar de instalar en el ordenador real?

Respuesta correcta: Porque si algo sale mal, solo borramos la VM y empezamos de nuevo.

💡 VirtualBox NO es más rápido que el metal real (al contrario, hay sobrecarga de virtualización). No es obligatorio por ley. La ventaja real es el aislamiento y la recuperación fácil: un error en la VM no daña el sistema anfitrión.

✅ Solución · Test rápido del alumno · B6

Pregunta: ¿Qué significa LTS en Ubuntu 24.04 LTS?

Respuesta correcta: Long Term Support: tiene 5 años de actualizaciones de seguridad.

💡 LTS no significa versión ligera ni es un acrónimo sin sentido. Las versiones no-LTS de Ubuntu tienen soporte de solo 9 meses. Para un entorno educativo o de trabajo, siempre se usa LTS.

📝 Notas didácticas · B1

Dificultad esperada: Los alumnos pueden confundir "multiusuario" con "varias cuentas en el mismo PC". Aclarar que un SO monousuario como MS-DOS ni siquiera tiene concepto de cuentas de usuario; todos comparten todo.

Error común: Pensar que Linux es un SO "solo para hackers" o "sin interfaz gráfica". Conviene mostrar Ubuntu con GNOME desde el primer día para romper este mito.

Pregunta trampa para debate: "¿Es Android un sistema operativo de escritorio?" Respuesta: No, está diseñado para pantallas táctiles y movilidad, aunque puede ejecutarse en tablets con teclado. Su kernel es Linux, pero no es una distribución GNU/Linux de escritorio.

✅ Solución · Refuerzo 1.2 · Identifica el tipo de SO

1. Brazo robótico en fábrica → Sistema operativo de tiempo real (RTOS). Debe garantizar que el movimiento del brazo ocurre exactamente cuando se ordena, sin retrasos.
2. Servidor con 50 empleados → SO multiusuario y multitarea (Windows Server, Ubuntu Server). Debe gestionar conexiones simultáneas.
3. Portátil antiguo con 2 GB RAM → SO ligero de 32 bits. Lubuntu, Linux Mint Xfce o Windows 7 32-bit si hay licencia.
4. Solo quiero software modificable → SO de código abierto. GNU/Linux en cualquiera de sus distribuciones libres.

✅ Solución · Profundización 1.2 · Debate software libre

Free Software Foundation (FSF): organización fundada en 1985 por Richard Stallman para promover el software libre. Defiende las 4 libertades: usar, estudiar, modificar y distribuir el software.

Richard Stallman: programador y activista estadounidense. Creó el proyecto GNU (1983), la licencia GPL y sentó las bases del movimiento del software libre.

Licencia GPL (General Public License): la licencia de software libre más usada. Exige que cualquier modificación del software también se distribuya como software libre (mecanismo "copyleft"). Linux, LibreOffice y la mayoría de herramientas del curso usan GPL.

Argumentos a favor del software libre en la Administración Pública: transparencia (el código puede auditarse), soberanía tecnológica (no dependes de Microsoft o Apple), ahorro en licencias, posibilidad de adaptar el software a necesidades locales, fomento de la industria local.

Argumentos a favor del software propietario: soporte técnico garantizado por contrato, integración con productos existentes, responsabilidad legal del fabricante, estándares de facto en muchas industrias.

✅ Solución · Test rápido del alumno · B8

Pregunta: ¿Qué extensión usa Writer (LibreOffice) para guardar documentos?

Respuesta correcta: A) .odt (OpenDocument Text).

💡 .docx es el formato de Microsoft Word (propietario). .pdf no es editable directamente. El formato nativo de LibreOffice Writer es .odt, aunque puede guardar en .docx para compatibilidad con Word.

✅ Solución · Test rápido del alumno · B9

Pregunta: ¿Qué comando crea una carpeta nueva?

Respuesta correcta: A) mkdir (en ambos SO, Windows y Linux).

💡 rmdir elimina carpetas vacías (remove directory). newdir no existe. mkdir = make directory, funciona igual en cmd.exe, PowerShell y bash.

✅ Solución · Test rápido del alumno · B10

Pregunta: ¿Qué atributo impide que un archivo sea modificado?

Respuesta correcta: A) Solo lectura (R).

💡 Oculto (H) solo esconde el archivo visualmente pero sigue siendo editable. Archivo (A) indica que el fichero ha sido modificado y pendiente de backup. Solo lectura (R) es el que bloquea la escritura.

✅ Solución · Test rápido del alumno · B11

Pregunta: ¿Qué permiso da chmod 600 archivo?

Respuesta correcta: Solo el dueño puede leer y escribir. Nadie más.

💡 600 en octal = rw------- : dueño tiene r(4)+w(2)=6, grupo y otros tienen 0. No es que todos puedan (eso sería 666) ni que nadie pueda (eso sería 000).

✅ Solución · Test rápido del alumno · B12

Pregunta: ¿Qué sistema de archivos NO permite guardar archivos de más de 4 GB?

Respuesta correcta: A) FAT32.

💡 NTFS no tiene ese límite práctico (soporta archivos de hasta 16 TB). exFAT fue diseñado específicamente para superar el límite de 4 GB de FAT32. FAT32 usa entradas de 32 bits para el tamaño del archivo, lo que limita el máximo a 4.294.967.295 bytes (≈ 4 GB).

✅ Solución · Test rápido del alumno · B13

Pregunta: Una impresora es un periférico de...

Respuesta correcta: A) Salida.

💡 La impresora recibe datos del ordenador y los convierte en papel impreso → solo salida. Un escáner sería entrada. Una pantalla táctil sería E/S.

✅ Solución · Test rápido del alumno · B14

Pregunta: ¿Qué tipo de malware secuestra tus archivos y te pide dinero?

Respuesta correcta: A) Ransomware.

💡 Un virus necesita un archivo huésped para propagarse. El adware muestra publicidad. El ransomware (ransom = rescate) cifra los archivos y exige pago para descifrarlos — WannaCry (2017) fue el ejemplo más famoso.

✅ Solución · Test rápido del alumno · B15

Pregunta: La estrategia 3-2-1 significa...

Respuesta correcta: 3 copias, 2 tipos de soporte, 1 copia fuera de casa/trabajo.

💡 No son 3 archivos ni intervalos de tiempo. La regla 3-2-1 garantiza que si tu casa se inunda o hay un incendio, la copia remota (nube, oficina de un familiar) sigue intacta.

📝 Notas didácticas · B2

Dificultad esperada: El concepto de "tiempo real" es difícil de entender porque los alumnos solo han usado sistemas de tiempo compartido. Conviene poner ejemplos de consecuencias catastróficas si un RTOS falla (accidente de avión, airbag que no se despliega).

Error común: "Si es gratis es libre, si es de pago es propietario". Falso. Hay software libre de pago (Red Hat Enterprise Linux) y software propietario gratuito (Google Chrome, Adobe Acrobat Reader).

B3

Solucionario · Arquitectura interna del SO: kernel, shell, sistema de archivos y drivers

⏱️ 110 min
✅ Solución · Ejercicio 1.3 · Identificación de capas del SO

1. Haces doble clic y se abre Writer → Kernel (gestión de procesos y memoria). El kernel localiza el ejecutable de Writer en el disco, le asigna un espacio en la RAM y lo lanza como un nuevo proceso.
2. Conectas un pendrive y aparece el Explorador → Kernel + Drivers. El driver USB detecta el dispositivo, informa al kernel, y el kernel notifica al shell (Explorador) que muestre la ventana.
3. Escribes dir o ls → Shell (CLI). El intérprete de comandos traduce la orden a llamadas al sistema que el kernel entiende.
4. "Necesitas permisos de administrador" → Kernel (gestión de seguridad). El kernel comprueba los permisos del archivo contra el usuario actual y deniega la operación.
5. Cambias el fondo de escritorio → Shell (GUI). La interfaz gráfica procesa tu acción y envía la orden de cambio al gestor de ventanas.

✅ Solución · Refuerzo 1.3 · Dibuja la arquitectura del SO

El diagrama debe mostrar las capas en este orden (de abajo arriba):
Hardware (CPU, RAM, disco, periféricos) → Kernel (gestión de procesos, memoria, E/S) → Drivers (controladores de dispositivos) y Sistema de archivos (NTFS/ext4) → Shell (GUI: escritorio, ventanas; CLI: cmd, bash) → Aplicaciones de usuario (LibreOffice, Firefox).

Cada capa solo se comunica con las capas inmediatamente adyacentes. Una aplicación no habla directamente con el hardware; siempre pasa por el kernel.

✅ Solución · Profundización 1.3 · Diferencias entre shells

cmd.exe: El shell tradicional de Windows, heredado de MS-DOS. Interpreta comandos simples como dir, copy, del. Muy limitado para scripting avanzado.

PowerShell: Shell moderno de Microsoft (2006). Orientado a objetos: en lugar de pasar texto entre comandos como bash, PowerShell pasa objetos .NET completos. Esto permite acceder a propiedades y métodos de los objetos directamente. Ejemplo: Get-Process | Where-Object {$_.CPU -gt 100}.

bash: Shell estándar en la mayoría de distribuciones Linux. Trabaja con flujos de texto usando pipes (|). Muy potente para procesar archivos de texto, logs y automatizar tareas del sistema.

zsh: Shell moderno para Linux/macOS, compatible con bash pero con mejoras: autocompletado más inteligente, temas personalizables, corrección ortográfica de comandos. Es el shell por defecto en macOS desde Catalina.

📝 Notas didácticas · B3

Dificultad esperada: El concepto de "kernel" es abstracto. Una analogía útil: el kernel es como el motor de un coche; el shell es el volante y los pedales. No ves el motor, pero sin él no funciona nada.

Demo impactante: Abrir el Administrador de tareas en Windows y mostrar la pestaña "Procesos". Todos esos procesos se están ejecutando "a la vez" gracias a la planificación del kernel. Luego abrir el Monitor del sistema en Ubuntu y hacer lo mismo.

B4

Solucionario · Arranque del ordenador: BIOS/UEFI, POST, bootloader y carga del SO

⏱️ 110 min
✅ Solución · Ejercicio 1.4 · Exploración de la BIOS/UEFI

Diferencias visuales BIOS vs UEFI: BIOS muestra una pantalla azul y gris con texto, navegable solo con teclado (Enter, flechas, Escape). UEFI muestra una interfaz gráfica más moderna, a menudo con el logo del fabricante, y permite usar el ratón.

Orden de arranque típico: 1º USB, 2º DVD/Unidad óptica, 3º Disco duro/SSD, 4º Red (PXE).

Secure Boot: En la mayoría de equipos de aula suele estar activado. Si está activado, solo arrancarán SO con bootloader firmado digitalmente (Windows, Ubuntu con Secure Boot soportado).

Virtualización: Es muy frecuente que en equipos de aula esté DESACTIVADA por defecto. Si es el caso, VirtualBox no podrá ejecutar máquinas virtuales de 64 bits o funcionará muy lento. Esto requerirá que el profesor o el servicio técnico la activen.

✅ Solución · Refuerzo 1.4 · Ordena la secuencia de arranque

Orden correcto:
1. Se pulsó el botón de encendido
2. Se ejecuta el firmware BIOS/UEFI
3. POST (Power On Self Test) — comprobación de hardware
4. Se elige el dispositivo de arranque según el orden configurado
5. El bootloader carga el kernel del SO
6. Se inicia el kernel y carga los drivers esenciales
7. Se inician los servicios en segundo plano
8. Aparece la pantalla de login

✅ Solución · Profundización 1.4 · Dual boot y reparación de arranque

Dual boot Windows + Ubuntu: Cuando instalas Ubuntu después de Windows, GRUB detecta automáticamente la instalación de Windows y añade una entrada en su menú. Al arrancar, GRUB te pregunta cuál de los dos sistemas quieres iniciar.

Partición EFI: En sistemas UEFI, existe una partición especial (normalmente de 100-500 MB, formateada en FAT32) que contiene los bootloaders de todos los SO instalados. Tanto Windows como Ubuntu guardan aquí sus gestores de arranque.

Problema común: Tras una actualización grande de Windows, este puede sobreescribir el bootloader por defecto y "olvidarse" de Ubuntu. El ordenador arranca directamente Windows sin mostrar el menú de GRUB.

Solución con Boot-Repair: 1) Arrancar desde un USB live de Ubuntu. 2) Instalar Boot-Repair: sudo add-apt-repository ppa:yannubuntu/boot-repair && sudo apt update && sudo apt install boot-repair. 3) Ejecutar Boot-Repair y elegir "Reparación recomendada". La herramienta reinstala GRUB y vuelve a detectar Windows.

📝 Notas didácticas · B4

Precaución: Antes de mandar al alumnado a explorar la BIOS/UEFI, asegúrate de que la BIOS no tiene contraseña puesta. Si la tiene, no podrán acceder o podrían bloquear el equipo.

Error común: "La BIOS es lo mismo que el SO". La BIOS/UEFI es firmware, no forma parte del SO. Se ejecuta antes que el SO y su única misión es preparar el hardware y ceder el control al bootloader.

B5

Solucionario · Instalación de Windows 11 en VirtualBox

⏱️ 110 min
✅ Solución · Ejercicio 1.5 · Documenta la instalación de Windows

1. Tiempo de instalación: Típicamente 20-40 minutos dependiendo de la velocidad del disco del equipo anfitrión. Si el disco es SSD, más cerca de 20 min; si es HDD, más cerca de 40 min.

2. Espacio ocupado por la VM: Recién instalado, el archivo .vdi ocupa entre 15 y 25 GB (aunque hayas asignado 60 GB, al ser dinámico solo ocupa lo que realmente se usa).

3. Opciones de privacidad: Windows 11 pregunta sobre: ubicación, diagnóstico, anuncios personalizados, sugerencias de contenido, actividad en la nube... Es buena práctica desactivar la mayoría durante la instalación y revisarlas después.

4. Cuenta Microsoft vs cuenta local: Windows 11 insiste en crear una cuenta Microsoft. Para crear una cuenta local: durante la instalación, en la pantalla de "inicia sesión", elegir "Opciones de inicio de sesión" → "Cuenta sin conexión" → "Experiencia limitada".

5. RAM consumida en reposo: Windows 11 recién instalado consume entre 2.5 y 3.5 GB de RAM sin abrir ningún programa adicional.

✅ Solución · Refuerzo 1.5 · Identifica errores de instalación

1. "No se encontró ningún dispositivo de arranque" → a) La ISO no está montada como unidad óptica en VirtualBox.
2. "Este equipo no cumple los requisitos mínimos" → b) La RAM asignada es inferior a 4 GB o el TPM no está habilitado.
3. "No se puede instalar Windows en este disco" → c) El disco está configurado como SSD NVMe sin driver. Cambiar a SATA en la configuración de la VM.
4. "La virtualización está desactivada" → d) VT-x/AMD-V no está activado en la BIOS.

✅ Solución · Profundización 1.5 · Instalación desatendida

Un archivo de respuesta (autounattend.xml) es un archivo XML que contiene todas las respuestas que el instalador de Windows haría durante una instalación interactiva (idioma, teclado, particionado, clave de producto, usuario...). Permite automatizar la instalación sin intervención humana.

Ventaja para 30 ordenadores: En lugar de sentarte 30 veces a responder las mismas preguntas, preparas un USB bootable con la ISO de Windows + el archivo autounattend.xml en la raíz. Arrancas los 30 equipos desde USB y se instalan solos.

Herramienta oficial: Windows System Image Manager (WSIM), incluido en el Windows ADK (Assessment and Deployment Kit). Es una herramienta gráfica que permite generar y validar archivos de respuesta sin necesidad de escribir XML a mano.

📝 Notas didácticas · B5

Problema frecuente: Alumnos que asignan solo 2 GB de RAM a la VM de Windows 11. No arranca o va extremadamente lento. Insistir en que 4 GB es el mínimo funcional.

Consejo: Si el aula tiene VirtualBox instalado pero la virtualización por hardware no está activada en BIOS, las VMs de 64 bits no funcionarán. Comprobadlo ANTES de esta sesión.

B6

Solucionario · Instalación de Ubuntu Linux y comparativa con Windows

⏱️ 110 min
✅ Solución · Ejercicio 1.6 · Comparativa de instalaciones

1. ¿Qué instalación fue más rápida? Ubuntu suele ser más rápido (15-25 min vs 20-40 min de Windows) porque tiene menos datos que copiar y no incluye tanta telemetría.

2. Pantalla de inicio de sesión: Windows muestra una imagen de fondo con la hora y el nombre del último usuario. Ubuntu (GNOME) muestra un fondo más minimalista con el nombre del usuario y un campo de contraseña. Ambas son funcionales pero estéticamente diferentes.

3. RAM consumida en reposo: Windows 11: 2.5-3.5 GB. Ubuntu 24.04 (GNOME): 1.2-1.8 GB. Ubuntu es más ligero. Procesos activos: Windows 150-200, Ubuntu 80-120.

4. Software preinstalado en Ubuntu que no viene en Windows: LibreOffice completo, Thunderbird, reproductor de vídeo, visor de PDF, utilidad de discos. En Windows que no viene en Ubuntu: Edge, Paint, Bloc de notas, Xbox, Cortana.

5. Creación de usuario: En Ubuntu es más simple: nombre, contraseña y ya. En Windows insiste en vincular a cuenta Microsoft (aunque se puede evitar con pasos adicionales).

✅ Solución · Refuerzo 1.6 · Completar la instalación

1. Writer debe abrirse y permitir guardar sin problemas. Si pide formato, usar .odt (formato nativo de Writer).
2. Firefox debe abrirse y cargar páginas normalmente. Ubuntu incluye Firefox como snap.
3. ls -la mostrará los archivos de la carpeta personal, incluyendo los ocultos (con punto).
4. df -h debería mostrar que el sistema ocupa entre 8 y 12 GB (mucho menos que Windows).

✅ Solución · Profundización 1.6 · Otras distribuciones Linux

Linux Mint: Basada en Ubuntu, interfaz Cinnamon (muy parecida a Windows en aspecto). Ideal para quien migra desde Windows. Incluye códecs multimedia de serie.
Fedora Workstation: Distribución de vanguardia patrocinada por Red Hat. Usa GNOME puro, paquetes muy actualizados. Más orientada a desarrolladores y entusiastas.
Debian: La "abuela" de Ubuntu. Extremadamente estable pero con software más antiguo. Ideal para servidores y equipos que no pueden permitirse fallos.

📝 Notas didácticas · B6

Momento "wow": Mostrar que Ubuntu ya viene con LibreOffice instalado y listo para usar. "Acabas de instalar el SO y ya puedes escribir documentos, hacer hojas de cálculo y presentaciones. En Windows tendrías que comprar Office o descargar algo adicional."

Error común: Olvidar la contraseña de Ubuntu. En Linux, la contraseña se usa para TODO (instalar, configurar, actualizar). Si el alumno la olvida, en una VM se puede reiniciar en modo recovery, pero es un engorro.

B7

Solucionario · Personalización del entorno, temas y accesibilidad

⏱️ 110 min
✅ Solución · Ejercicio 1.7 · Personaliza tu entorno

B8

Solucionario · Sistema de archivos: rutas, extensiones, estructura

⏱️ 110 min
✅ Solución · Ejercicio 1.8 · Rutas y extensiones

Rutas absolutas típicas:
Windows: Documentos → C:\Users\[usuario]\Documents, Descargas → C:\Users\[usuario]\Downloads, Escritorio → C:\Users\[usuario]\Desktop
Ubuntu: Documentos → /home/[usuario]/Documentos, Descargas → /home/[usuario]/Descargas, Escritorio → /home/[usuario]/Escritorio

Ruta absoluta vs relativa: Absoluta empieza desde la raíz (C:\ o /). Relativa empieza desde la posición actual. Ejemplo: si estás en /home/alumno/, la ruta relativa Documentos/archivo.txt equivale a la absoluta /home/alumno/Documentos/archivo.txt.

Extensiones de LibreOffice: Writer → .odt, Calc → .ods, Impress → .odp.

✅ Solución · Refuerzo 1.8 · Identifica tipos de archivo

1. presentacion.odp → Presentación de LibreOffice Impress
2. foto_vacaciones.jpg → Imagen (se abre con Visor de imágenes)
3. presupuesto_2025.ods → Hoja de cálculo de LibreOffice Calc
4. carta_cliente.odt → Documento de texto de LibreOffice Writer
5. musica_fondo.mp3 → Archivo de audio (reproductor multimedia)
6. comprimido.zip → Archivo comprimido (se abre con el gestor de archivos)

✅ Solución · Profundización 1.8 · Permisos en Linux

-rwxr-xr-- se descompone así:
- Primer guión: es un archivo normal (no directorio).
- rwx (dueño): lectura, escritura y ejecución → octal 7
- r-x (grupo): lectura y ejecución, sin escritura → octal 5
- r-- (otros): solo lectura → octal 4
Octal equivalente: 754

Para que solo el dueño pueda leer y escribir: chmod 600 archivo (dueño: rw-, grupo y otros: sin permisos).

📝 Notas didácticas · B8

Error muy común: Los alumnos confunden las barras de Windows (\) con las de Linux (/). En Windows, cd / no funciona; en Linux, cd \ no funciona. Conviene practicar esto explícitamente.

✅ Solución · Refuerzo 1.9 · Comandos básicos de terminal

1. dir → Lista archivos y carpetas en el directorio actual (Windows)
2. ls -la → Lista archivos incluyendo ocultos, con detalles (Linux)
3. cd .. → Sube al directorio padre (ambos)
4. mkdir practica → Crea una carpeta llamada practica (ambos)
5. rm -rf carpeta → Borra una carpeta y todo su contenido sin preguntar (Linux, muy peligroso)
6. copy a.txt b.txt → Copia a.txt a b.txt (Windows)

✅ Solución · Profundización 1.9 · Scripts de automatización

Script para Windows (.bat):

@echo off
mkdir "%USERPROFILE%\Desktop\Backup"
copy C:\Origen\*.txt "%USERPROFILE%\Desktop\Backup\"
echo Copia finalizada
pause

Script para Linux (.sh):

#!/bin/bash
mkdir -p ~/Escritorio/Backup
cp /ruta/origen/*.txt ~/Escritorio/Backup/
echo "Copia finalizada"

El script de Linux debe tener permisos de ejecución: chmod +x script.sh.

Momento crítico: Cuando los alumnos usan rm en Linux por primera vez. Insistir en que NO hay papelera en la terminal. Borrado = permanente. Recomendar rm -i para que pregunte antes de borrar.

B10

Solucionario · Atributos de archivos: solo lectura, oculto, sistema

⏱️ 110 min
✅ Solución · Ejercicio 1.10 · Experimenta con atributos

2. Solo lectura: Al intentar guardar cambios, el Bloc de notas (o cualquier editor) mostrará "Acceso denegado" o sugerirá "Guardar como" con otro nombre. No puedes sobrescribir un archivo de solo lectura.

3. Atributo oculto: Sí, el archivo desaparece visualmente del escritorio. Para verlo: en el Explorador → pestaña Vista → marcar "Elementos ocultos".

4. attrib importante.txt: Si tiene solo lectura y oculto, mostrará: A RH C:\Users\.... R = Read-only, H = Hidden. La A = Archive (está pendiente de backup porque fue modificado).

5. Después de attrib -r -h importante.txt, el archivo vuelve a ser visible y editable.

✅ Solución · Refuerzo 1.10 · Identifica atributos

1. Archivo de configuración consultable pero no modificable → Solo lectura (R)
2. Archivo del sistema que no debe verse ni tocarse → Oculto (H) + Sistema (S)
3. Documento modificado que necesita backup → Archivo (A). Windows lo activa automáticamente al modificar el archivo.

✅ Solución · Profundización 1.10 · ADS en NTFS

Los Alternate Data Streams (ADS) son una característica de NTFS que permite adjuntar datos ocultos a un archivo sin modificar su contenido visible. Por ejemplo: echo "Hola" > archivo.txt:oculto.txt crea un flujo alternativo llamado oculto.txt dentro de archivo.txt. El archivo parece normal, pero tiene datos escondidos.

Para leerlo: more < archivo.txt:oculto.txt o notepad archivo.txt:oculto.txt.

Relevancia seguridad: Los ADS pueden ser usados por malware para ocultar código malicioso. Un antivirus debe escanear también los flujos alternativos. Para listar flujos: dir /r en cmd.

B11

Solucionario · Permisos en Windows (NTFS) y Linux (chmod)

⏱️ 110 min
✅ Solución · Ejercicio 1.11 · Configura permisos

Windows: Al quitar todos los permisos al usuario Invitado en la carpeta Privada, al iniciar sesión como ese usuario e intentar acceder a C:\Privada\, Windows mostrará: "No tiene permiso para acceder a esta carpeta". Si se quitaron incluso los permisos de "Listar contenido de carpeta", la carpeta ni siquiera será visible.

Ubuntu: Después de chmod 600 ~/confidencial.txt, ls -l mostrará -rw-------. Solo el dueño puede leer y escribir. Si otro usuario intenta leerlo (sudo -u otro cat ~/confidencial.txt), obtendrá "Permission denied".

✅ Solución · Refuerzo 1.11 · Calcula permisos octales

1. rwxr-xr-x → 755 (rwx=7, r-x=5, r-x=5)
2. rw-r--r-- → 644 (rw-=6, r--=4, r--=4)
3. rwx------ → 700 (rwx=7, ---=0, ---=0)
4. chmod 754 → rwxr-xr-- (dueño rwx, grupo r-x, otros r--)

✅ Solución · Profundización 1.11 · Permisos especiales

SUID (Set User ID, octal 4---): Un archivo con SUID se ejecuta con los privilegios del PROPIETARIO del archivo, no del usuario que lo ejecuta. Ejemplo: /usr/bin/passwd pertenece a root, pero cualquier usuario puede cambiar su propia contraseña porque passwd tiene SUID activo. Sin SUID, un usuario normal no podría modificar /etc/shadow.

SGID (Set Group ID, octal 2---): Similar pero para grupos. En una carpeta con SGID, los archivos creados dentro heredan el grupo de la carpeta, no el grupo del usuario que los crea. Muy útil en carpetas compartidas de trabajo en equipo.

Sticky Bit (octal 1---): En una carpeta como /tmp, todos pueden crear archivos, pero solo el PROPIETARIO puede borrarlos, aunque otros tengan permiso de escritura en la carpeta. Sin Sticky Bit, cualquier usuario podría borrar archivos de otros en /tmp.

B12

Solucionario · Exploración de soportes de almacenamiento

⏱️ 110 min
✅ Solución · Ejercicio 1.12 · Administración de almacenamiento

Windows: El Administrador de discos mostrará normalmente 1 disco (Disco 0) con 2-3 particiones (recuperación, arranque EFI, C:). C: suele ocupar la mayor parte del disco.

Ubuntu: lsblk -f mostrará los discos con sus sistemas de archivos (ext4 para la raíz, vfat para EFI). df -h mostrará el espacio usado y disponible.

FAT32: El tamaño máximo de archivo individual en FAT32 es 4 GB. No podrás guardar archivos mayores que eso, aunque el disco tenga espacio libre.

✅ Solución · Refuerzo 1.12 · Elige el sistema de archivos

1. Pendrive 64 GB para Windows/Mac/Linux → exFAT. Es compatible con los tres SO y no tiene el límite de 4 GB de FAT32.
2. Disco para Ubuntu → ext4. Es el sistema de archivos nativo de Linux, ofrece buen rendimiento y journaling.
3. Disco externo 4 TB para backups en Windows → NTFS. Soporta archivos enormes, tiene permisos y journaling. Para backups es ideal.
4. Tarjeta SD de cámara antigua → FAT32. Muchas cámaras antiguas solo reconocen FAT32.

✅ Solución · Profundización 1.12 · RAID

RAID 0 (striping): Divide los datos entre 2 o más discos. Mejora la velocidad pero no hay redundancia. Si falla UN disco, se pierde TODO. Solo para velocidad.

RAID 1 (mirroring): Duplica los datos en 2 discos idénticos. Si falla uno, el otro sigue funcionando. Capacidad = la del disco más pequeño. Ideal para el SO.

RAID 5 (striping con paridad): Necesita mínimo 3 discos. Divide los datos y guarda información de paridad para reconstruir lo perdido si falla UN disco. Buen equilibrio velocidad/capacidad/redundancia. Muy usado en servidores.

Solucionario · Periféricos: drivers, administrador de dispositivos

⏱️ 110 min
✅ Solución · Ejercicio 1.13 · Identificación de periféricos y drivers

Los alumnos deben identificar al menos 10 periféricos en el Administrador de dispositivos. La clasificación esperada es: teclado (entrada), ratón (entrada), monitor (salida), disco duro (E/S), tarjeta de red (E/S), tarjeta gráfica (salida), webcam (entrada), micrófono (entrada), altavoces (salida), impresora (salida).

Driver de la tarjeta gráfica: Si es una VM de VirtualBox, el driver será "VirtualBox Graphics Adapter". La fecha y versión dependerán de la versión de VirtualBox y Guest Additions instaladas.

✅ Solución · Refuerzo 1.13 · Clasifica periféricos

1. Escáner → Entrada
2. Proyector → Salida
3. Pantalla táctil → Entrada/Salida (muestra imágenes y recibe toques)
4. Cascos con micrófono → Entrada/Salida (altavoces de salida, micrófono de entrada)
5. Lector de código de barras → Entrada

✅ Solución · Profundización 1.13 · Drivers genéricos vs fabricante

Driver genérico: incluido en Windows. Proporciona funcionalidad básica y suele ser estable. Ejemplo: el driver de impresora "HP Universal Print Driver" permite imprimir en casi cualquier HP, pero solo imprime, no escanea.

Driver del fabricante: descargado de la web oficial. Ofrece todas las funciones, mejor rendimiento y a menudo herramientas adicionales. Ejemplo: el driver específico de una impresora HP multifunción permite escanear, ver niveles de tinta y usar funciones avanzadas.

B14

Solucionario · Antivirus y protección contra malware

⏱️ 110 min
✅ Solución · Ejercicio 1.14 · Configura tu protección antivirus

Windows: El examen rápido suele tardar 1-3 minutos y analizar 20.000-50.000 archivos. Normalmente no encuentra amenazas (a menos que el equipo esté infectado).

Ubuntu: ClamAV en una instalación limpia de Ubuntu no suele encontrar nada. El escaneo debería devolver "0 infected files".

Test EICAR: Windows Defender debe detectar el archivo eicar.com inmediatamente al guardarlo (o incluso antes, al intentar crearlo) y ponerlo en cuarentena. Esto demuestra que el antivirus está funcionando en tiempo real.

✅ Solución · Refuerzo 1.14 · Identifica el tipo de amenaza

1. Cifra documentos y pide rescate → Ransomware
2. Muestra publicidad invasiva → Adware
3. Se propaga solo por la red → Gusano (Worm)

✅ Solución · Profundización 1.14 · Caso WannaCry

WannaCry (mayo 2017): Ataque de ransomware que afectó a más de 200.000 equipos en 150 países. Se propagaba automáticamente como un gusano usando la vulnerabilidad EternalBlue (MS17-010) en el protocolo SMB de Windows, supuestamente desarrollada por la NSA y filtrada por el grupo Shadow Brokers.

Sectores más afectados: NHS (sanidad británica), Telefónica, Renault, FedEx. Muchos hospitales tuvieron que cancelar operaciones.

Cómo se detuvo: El investigador Marcus Hutchins (@MalwareTech) descubrió que el malware intentaba conectar con un dominio no registrado. Al registrar ese dominio, activó accidentalmente un "kill switch" que detuvo la propagación.

¿Podría repetirse? Sí. Aunque EternalBlue fue parcheado, constantemente se descubren nuevas vulnerabilidades. La defensa: mantener sistemas actualizados, backups offline y segmentación de red.

📝 Notas didácticas · B14

Cuidado con el archivo EICAR: Algunos antivirus pueden alarmarse. Explicar antes que es INOFENSIVO, es un test estándar de la industria desde los años 90. Aún así, puede que el antivirus de algunos portátiles personales lo detecte y los alumnos se asusten.

B15

Solucionario · Copias de seguridad y restauración

⏱️ 110 min
✅ Solución · Ejercicio 1.15 · Configura copias de seguridad

Windows: El Historial de archivos debe mostrar la copia realizada. Al restaurar doc3.txt, el archivo aparece de nuevo en su ubicación original con su contenido intacto.

Ubuntu: Déjà Dup debe completar el backup sin errores y permitir restaurar el archivo borrado.

✅ Solución · Refuerzo 1.15 · Planifica tu estrategia de backup

Fotógrafo (500 GB): Estrategia 3-2-1: (1) original en el PC de trabajo, (2) backup diario incremental a disco externo de 2 TB, (3) backup semanal a NAS en otra habitación, (4) backup mensual completo a la nube (Backblaze, pCloud). Las fotos de bodas son irreemplazables: no hay "segunda oportunidad" para capturar ese momento.

Estudiante (apuntes): (1) original en el portátil, (2) backup automático a la memoria USB de 32 GB al final de cada clase, (3) copia semanal en Google Drive o OneDrive gratuito (suficiente para documentos pequeños).

✅ Solución · Profundización 1.15 · Cifrado de copias de seguridad

¿Por qué cifrar? Si subes tus copias de seguridad a la nube sin cifrar, el proveedor (Google, Microsoft, Dropbox) puede técnicamente acceder a tus archivos. Si hay una brecha de seguridad o una solicitud gubernamental, tus datos quedan expuestos.

VeraCrypt: crea "volúmenes cifrados" (archivos grandes que actúan como discos virtuales). Todo lo que guardas dentro está cifrado con AES-256. Para la nube, subes el archivo del volumen. Necesitas descargarlo entero para descifrar un solo archivo.

Cryptomator: cifra archivo por archivo antes de subirlos a la nube. Es transparente: ves tus archivos normalmente en una carpeta virtual, pero en Google Drive/OneDrive solo se ven archivos con nombres aleatorios y contenido cifrado. Mejor para backups en la nube porque no necesitas descargar todo el volumen.
