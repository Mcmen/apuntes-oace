🧪 RA1 – Prácticas de Laboratorio · 16 Prácticas Evaluables

Cada práctica se corresponde con un bloque de los apuntes del alumno. Todas las rúbricas están diseñadas con criterios observables y medibles para que puedan ser corregidas por una IA sin ambigüedad.

📦 Entorno de trabajo: Oracle VirtualBox con Windows 11 y Ubuntu 24.04 LTS como máquinas virtuales invitadas.

📦 Cómo entregar las prácticas

Todas las prácticas de RA1 se entregan en una única carpeta comprimida. El sistema de corrección automática con IA leerá los archivos por su nombre exacto, por lo que es imprescindible respetar la nomenclatura al pie de la letra.

1. Estructura de carpetas y nombre del ZIP

Un único ZIP con todas las prácticas de RA1. La estructura interna debe ser exactamente así:

Garcia_Juan_RA1.zip └── Garcia_Juan/ ├── P1-1/ │ └── P1-1_tabla.pdf ├── P1-2/ │ └── P1-2_tabla.pdf ├── P1-3/ │ ├── P1-3_documento.pdf │ ├── P1-3_captura01.png │ └── P1-3_captura08.png ├── ... └── P1-16/ ├── P1-16_documento.pdf └── CONFIGURACION.txt

2. Reglas de nomenclatura

ElementoFormatoEjemplo
ZIP principalApellido_Nombre_RA1.zipGarcia_Juan_RA1.zip
Carpeta raíz dentro del ZIPApellido_Nombre/Garcia_Juan/

ℹ️ Apellido y nombre sin tildes ni ñ. Si tienes dos apellidos, usa solo el primero. Guion bajo (_), nunca espacios.

3.. Plazo y canal de entrega

El ZIP de RA1 completo se entrega en la tarea habilitada en el aula virtual (Moodle) antes del fin del bloque 16. No se aceptan entregas por correo electrónico ni por USB. Si tienes problemas técnicos para crear el ZIP, comunícaselo al profesor antes de la fecha límite.

Arquitectura del SO: identificación práctica de capas

⏱️ 55 min 📘 Bloque 3
RA1 CE-h · Conceptos básicos SO (15%) RA1 CE-c · Usar interfaz (5%)
🎯 Objetivos
  • Identificar las capas del sistema operativo en funcionamiento real.
  • Usar comandos de terminal para obtener información del kernel, shell y sistema de archivos.
  • Comparar la arquitectura interna de Windows y Linux.
🔨 Tareas a realizar

En Windows (VM):

  1. Abre PowerShell como administrador. Ejecuta systeminfo | findstr /B /C:"OS" para ver la versión del SO.
  2. Ejecuta Get-ComputerInfo | Select-Object WindowsVersion, OsHardwareAbstractionLayer para ver información del kernel.
  3. Abre el Administrador de dispositivos (devmgmt.msc) y captura la lista de drivers de la categoría "Procesadores".
  4. Ejecuta fsutil fsinfo volumeinfo C: para ver información del sistema de archivos.

En Ubuntu (VM):

  1. Ejecuta uname -a para ver la versión del kernel Linux.
  2. Ejecuta echo $SHELL para ver qué shell estás usando.
  3. Ejecuta lsblk -f para ver los sistemas de archivos montados.
  4. Ejecuta lspci | head -10 para ver dispositivos PCI (drivers cargados).

Análisis: Para cada comando ejecutado, identifica a qué capa del SO pertenece la información obtenida (kernel, shell, sistema de archivos o drivers).

P1.4

Exploración de la BIOS/UEFI, bootloader y secuencia de arranque

⏱️ 55 min 📘 Bloque 4
RA1 CE-h · Conceptos básicos SO (15%) RA1 CE-a · Parámetros instalación (10%)
🎯 Objetivos
  • Explorar la configuración de la BIOS/UEFI de un equipo real (sin modificar nada).
  • Identificar el bootloader en ambos sistemas operativos.
  • Documentar la secuencia de arranque completa del equipo del aula.
🔨 Tareas a realizar

⚠️ ADVERTENCIA: No modifiques NINGUNA opción en la BIOS/UEFI. Solo observar y anotar.

  1. Reinicia el ordenador del aula y accede a la BIOS/UEFI (consulta a tu profesor la tecla correcta: F2, Supr, F10, F12...).
  2. Identifica si es BIOS (pantalla azul-gris, solo teclado) o UEFI (interfaz gráfica, ratón). Anótalo.
  3. Localiza y anota: versión del firmware, fecha de la BIOS/UEFI, orden de arranque (boot order) y estado de Secure Boot.
  4. Busca la opción de virtualización (VT-x en Intel o AMD-V en AMD) y anota si está activada o desactivada.
  5. Sal sin guardar cambios.
  6. En Windows, abre una terminal como administrador y ejecuta bcdedit /enum. Identifica la entrada del bootloader de Windows y captura la salida.
  7. En Ubuntu, ejecuta cat /boot/grub/grub.cfg | head -30 para ver la configuración de GRUB (no la modifiques).
  8. Dibuja un esquema de la secuencia de arranque de tu equipo desde que pulsas el botón hasta que aparece el login.
📦 Entrega

📁 Carpeta: Apellido_Nombre/P1-4/

  • P1-4_documento.pdf — Datos anotados de BIOS/UEFI + esquema de la secuencia de arranque
  • P1-4_bios.jpg — Foto o captura de la pantalla de BIOS/UEFI
  • P1-4_bcdedit.png — Captura de bcdedit en Windows
  • P1-4_grub.png — Captura del menú GRUB en Ubuntu

📊 Rúbrica de corrección · P1.4

  • Crear una máquina virtual correctamente dimensionada para Windows 11.
  • Instalar Windows 11 desde cero documentando cada decisión.
  • Realizar la configuración inicial básica del sistema recén instalado.
🔨 Tareas a realizar
  1. Crear la VM: Nombre: W11-P5-[tus_iniciales]. Tipo: Microsoft Windows. Versión: Windows 11 (64-bit). Asigna 4096 MB de RAM, 2 CPUs, disco VDI dinámico de 60 GB.

  5. Documenta el proceso: captura de pantalla de la configuración de la VM, de la pantalla de selección de idioma, de la creación de la cuenta local y del escritorio final.

P1.9

Operaciones con archivos: GUI vs Terminal

⏱️ 90 min 📘 Bloque 9
RA1 CE-e · Manejo de archivos (10%) RA1 CE-f · Herramientas del SO para explorar soportes (10%)
🎯 Objetivos
  • Realizar las mismas operaciones con archivos usando GUI y CLI.
  • Comparar la eficiencia de ambos métodos.
  • Dominar los comandos básicos de terminal en ambos SO.
🔨 Tareas a realizar

Vas a crear la MISMA estructura de archivos DOS VECES: una usando solo la GUI y otra usando solo la terminal. Hazlo en ambos SO (4 veces en total: Windows GUI, Windows CLI, Ubuntu GUI, Ubuntu CLI).

  1. Crea esta estructura (el nombre de la carpeta raíz debe ser PRUEBA en el escritorio): 
    PRUEBA/
├── Documentos/
│   ├── Notas/
│   └── Facturas/
├── Imagenes/
│   ├── Fotos/
│   └── Capturas/
└── Musica/
    ├── Pop/
    └── Rock/
  2. Dentro de Documentos/Notas/, crea un archivo recordatorio.txt con el texto "Reunión el lunes a las 10:00".
  3. Copia recordatorio.txt a la carpeta Facturas/ y renómbralo como backup.txt.

🔨 Tareas a realizar

  3. Escanea tu carpeta personal: clamscan -r ~/ (anota el resultado).
  4. Vuelve a crear el archivo EICAR en Ubuntu y escanea la carpeta donde lo guardaste. ¿Lo detecta ClamAV?
