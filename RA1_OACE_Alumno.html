B1

¿Qué es un sistema operativo? Funciones, tipos y ejemplos

⏱️ 110 min
RA1 CE-h · Conceptos básicos SO (15%)

Un sistema operativo (SO) es el conjunto de programas que gestionan los recursos físicos (hardware) de un ordenador y proporcionan una plataforma sobre la que se ejecutan el resto de aplicaciones. Sin un sistema operativo, un ordenador no puede funcionar de forma útil para el usuario medio. El SO actúa como intermediario entre el hardware y las aplicaciones: traduce las peticiones de los programas en instrucciones que el hardware puede ejecutar, y gestiona quién usa cada recurso en cada momento para evitar conflictos.

📌 Definición técnica
El sistema operativo es la capa de software que se sitúa entre el hardware y las aplicaciones de usuario. Se carga al encender el ordenador y permanece en memoria durante todo el funcionamiento. Sus dos objetivos principales son: (1) hacer que el ordenador sea utilizable por personas sin conocimientos técnicos profundos, y (2) aprovechar los recursos hardware de forma eficiente.

🧩 Las seis funciones esenciales de cualquier SO:

⚙️ 1. Gestión del procesador (CPU)

El SO decide qué programa usa el procesador en cada instante mediante la planificación de procesos. Como la CPU ejecuta miles de millones de operaciones por segundo, el SO reparte el tiempo entre todos los programas activos, dando la sensación de que funcionan simultáneamente (multitarea). Cada programa recibe un pequeño intervalo de tiempo y luego cede el turno al siguiente.

🧠 2. Gestión de la memoria (RAM)

Asigna espacio en la RAM a cada programa. Cuando la RAM se llena, el SO usa una parte del disco como memoria virtual (archivo de paginación en Windows, swap en Linux). También libera la memoria cuando un programa se cierra y protege a cada proceso para que no pueda acceder a la memoria de otros (aislamiento de procesos).

💾 3. Gestión del almacenamiento

Organiza los datos en archivos y directorios sobre discos duros, SSD, USB, etc. El sistema de archivos (NTFS, ext4, FAT32, exFAT) define las reglas para guardar, leer, modificar y borrar información de forma fiable, incluyendo mecanismos de recuperación ante fallos.

🖨️ 4. Gestión de dispositivos (E/S)

Controla todos los periféricos mediante drivers (controladores), pequeños módulos de software que enseñan al SO cómo comunicarse con cada dispositivo concreto. Cuando conectas un pendrive, la impresora o unos auriculares Bluetooth, es el SO quien detecta el dispositivo y carga el driver adecuado.

🔐 5. Gestión de usuarios y seguridad

Permite crear diferentes cuentas con distintos niveles de privilegios. El administrador puede instalar software y cambiar configuraciones del sistema; un usuario estándar solo puede usar programas ya instalados y acceder a sus propios archivos. Esto evita que software malicioso o errores de usuario dañen el sistema.

🖥️ 6. Interfaz de usuario

Proporciona la forma de interactuar con el ordenador. La GUI (Graphical User Interface) usa ventanas, iconos, menús y ratón; la CLI (Command Line Interface) se basa en escribir comandos en una terminal. La mayoría de SO modernos ofrecen ambas: el usuario normal usa la GUI, y el técnico avanzado recurre a la CLI para tareas potentes y automatización.

B2

Tipos de SO: monousuario/multiusuario, monotarea/multitarea, 32/64 bits, tiempo real, libre/propietario

⏱️ 110 min
RA1 CE-h · Conceptos básicos SO (15%)

No todos los sistemas operativos funcionan igual ni están diseñados para lo mismo. Clasificarlos según diferentes criterios ayuda a entender por qué elegimos uno u otro según el trabajo que vayamos a hacer y el hardware disponible.

👤 Según el número de usuarios

Monousuario: Solo una persona puede usar el sistema a la vez. Ejemplos: MS-DOS, Windows 95/98 (no permitían cambiar de usuario sin reiniciar).
Multiusuario: Varias personas pueden tener sesiones abiertas simultáneamente, cada una con su propia configuración, archivos privados y permisos independientes. Todos los SO modernos (Windows 11, Ubuntu, macOS) son multiusuario.

⏱️ Según el número de tareas

Monotarea: Solo ejecuta un programa a la vez. Hasta que no cierras una aplicación, no puedes abrir otra. Ejemplo: MS-DOS original.
Multitarea: Ejecuta varios programas simultáneamente repartiendo el tiempo de CPU entre ellos. Puedes escribir en Writer mientras se descarga un archivo y suena música de fondo. Todos los SO actuales son multitarea.

🔢 Según la arquitectura

32 bits (x86): Puede direccionar hasta 4 GB de RAM como máximo. Hoy en día está casi extinguido en ordenadores de escritorio, pero aún se usa en dispositivos embebidos y móviles antiguos.
64 bits (x64): Puede manejar muchísima más RAM (terabytes teóricos). Es más rápido para aplicaciones pesadas y es el estándar en cualquier ordenador moderno.

⏲️ Según el tiempo de respuesta

Tiempo compartido: El SO reparte el tiempo de CPU entre todos los procesos sin garantizar cuándo terminará cada uno. Si hay mucha carga, todo va más lento. Es el modelo de Windows, Linux, macOS.
Tiempo real: El SO garantiza que ciertas tareas críticas se ejecutan en un plazo máximo de tiempo. Si no se cumple, puede haber consecuencias graves. Se usa en aviones, marcapasos, frenos ABS, maquinaria industrial.

🔓 Software libre vs software propietario:

🔓 Software libre (open source)

  • El código fuente está disponible públicamente.
  • Cualquiera puede estudiarlo, modificarlo y distribuir cambios.
  • Suele ser gratuito (aunque "libre" no significa necesariamente "gratis").
  • Ejemplos: Linux, LibreOffice, Firefox, VLC, GIMP, VirtualBox, Blender.
  • Ventaja: comunidad global que corrige errores y añade mejoras. No dependes de una sola empresa.

🔒 Software propietario

  • El código fuente es secreto de la empresa creadora.
  • Solo la empresa puede modificarlo o distribuir copias.
  • Suele ser de pago (aunque hay versiones gratuitas limitadas).
  • Ejemplos: Windows, Microsoft Office, Adobe Photoshop, AutoCAD.
  • Ventaja: soporte técnico oficial, garantía de la empresa, integración con otros productos.
🔑 Concepto clave

"Software libre" no significa necesariamente "gratis". Significa que tienes libertad para usar, estudiar, modificar y compartir el software. El software libre puede ser de pago (aunque lo más habitual es que también sea gratuito).

B3

Arquitectura interna del SO: kernel, shell, sistema de archivos y drivers

⏱️ 110 min
RA1 CE-h · Conceptos básicos SO (15%) RA1 CE-c · Usar interfaz (5%)

Un sistema operativo no es un bloque monolítico, sino una estructura organizada en capas que trabajan de forma coordinada. Comprender esta arquitectura es esencial para cualquier técnico informático, porque permite entender dónde se produce un fallo y cómo interactúan los distintos componentes del sistema.

📌 Las cuatro capas fundamentales de un SO moderno

1. Kernel (núcleo): El corazón del SO. Es lo primero que se carga al encender el ordenador y permanece en memoria permanentemente. Gestiona directamente la CPU, la RAM, los procesos y la comunicación con los dispositivos hardware. Es la única parte del SO que interactúa directamente con el hardware. Windows usa el kernel NT; Linux usa el kernel Linux (creado por Linus Torvalds en 1991).

2. Shell (intérprete de comandos o capa de interfaz): La capa que permite al usuario comunicarse con el kernel. Puede ser gráfica (el escritorio de Windows, GNOME en Ubuntu) o de texto (cmd.exe, PowerShell, bash en Linux). Traduce las órdenes del usuario a llamadas que el kernel entiende.

3. Sistema de archivos: Define cómo se organizan, nombran, almacenan y recuperan los datos en los discos. Establece la estructura de directorios (carpetas), los nombres de archivo, los permisos de acceso y los mecanismos de recuperación ante fallos. Ejemplos: NTFS (Windows), ext4 (Linux), APFS (macOS).

4. Drivers (controladores): Pequeños módulos de software que enseñan al kernel cómo comunicarse con cada dispositivo específico. Sin el driver adecuado, el dispositivo no funciona o lo hace de forma limitada. Ejemplo: el driver de una impresora HP le dice al kernel exactamente qué comandos enviar para imprimir correctamente.

🔍 Comparativa: kernel de Windows (NT) vs kernel de Linux

🪟 Kernel NT (Windows)

  • Desarrollado por Microsoft desde 1993
  • Código cerrado, solo Microsoft puede modificarlo
  • Usado en todas las versiones desde Windows 2000
  • Los drivers son específicos de cada versión
  • Actualizaciones a través de Windows Update
  • Soporta arquitecturas x86, x64 y ARM64

🐧 Kernel Linux

  • Creado por Linus Torvalds en 1991
  • Código abierto, mantenido por miles de desarrolladores
  • Usado en Linux, Android, ChromeOS, servidores, supercomputadores
  • Muchos drivers vienen integrados en el propio kernel
  • Se actualiza independientemente de la distribución
  • Soporta más de 20 arquitecturas diferentes

🖥️ GUI vs CLI: dos formas de hablar con el ordenador

  • GUI (Graphical User Interface): Interfaz gráfica con ventanas, iconos, menús y puntero del ratón. Es la que usas al encender Windows (escritorio con barra de tareas) o Ubuntu (entorno GNOME). Es intuitiva pero limitada para tareas masivas o automatizadas.
  • CLI (Command Line Interface): Interfaz de línea de comandos. Escribes órdenes de texto y el sistema responde. Mucho más potente para tareas técnicas, administración de sistemas y automatización. Ejemplos: cmd.exe y PowerShell (Windows), bash y zsh (Linux).
🔑 Concepto clave

El kernel es la única parte del SO que realmente interactúa con el hardware. Todo lo demás (shell, programas, drivers) se comunica con el kernel, que actúa como un intermediario universal entre el software y el hardware físico.

B4

Arranque del ordenador: BIOS/UEFI, POST, bootloader y carga del SO

⏱️ 110 min
RA1 CE-h · Conceptos básicos SO (15%) RA1 CE-a · Parámetros básicos instalación (10%)

Cuando pulsas el botón de encendido, ocurre una secuencia muy precisa de acontecimientos antes de que aparezca el escritorio. Entender este proceso es fundamental para diagnosticar problemas de arranque, instalar sistemas operativos y configurar el orden de inicio de los dispositivos.

📌 La secuencia de arranque completa paso a paso
  1. Alimentación: la fuente de alimentación proporciona corriente a la placa base, CPU, RAM y discos.
  2. Firmware (BIOS/UEFI): se ejecuta el programa almacenado en un chip de memoria flash de la placa base.
  3. POST (Power On Self Test): comprueba que CPU, RAM, tarjeta gráfica, teclado y demás componentes esenciales funcionan. Si algo falla, emite una secuencia de pitidos de error o muestra un código.
  4. Detección de dispositivos: identifica los discos duros, SSD, unidades USB, lectoras de DVD y tarjetas de red conectados.
  5. Selección del dispositivo de arranque: según el orden configurado en la BIOS/UEFI (por ejemplo: 1º USB, 2º DVD, 3º disco duro), elige desde cuál iniciar.
  6. Lectura del sector de arranque: lee el primer sector del disco elegido, que contiene el bootloader.
  7. Bootloader: el gestor de arranque localiza el kernel del SO en el disco y lo carga en la memoria RAM. Windows usa Windows Boot Manager; Linux suele usar GRUB.
  8. Carga del kernel: el SO toma el control, inicializa sus estructuras internas y carga los drivers esenciales.
  9. Servicios: se inician los procesos y servicios en segundo plano (red, sonido, firewall, actualizaciones...).
  10. Login: aparece la pantalla de inicio de sesión para que el usuario introduzca sus credenciales.

🔧 BIOS vs UEFI: el firmware de la placa base

BIOS (Basic Input/Output System)

  • Tecnología de los años 80, ya obsoleta
  • Interfaz de texto azul y gris, solo teclado
  • Usa tabla de particiones MBR (Master Boot Record)
  • Solo soporta discos de hasta 2 TB de capacidad
  • Máximo 4 particiones primarias por disco físico
  • Arranque más lento (varios segundos)
  • Vulnerable a malware que sobreescribe el sector de arranque

UEFI (Unified Extensible Firmware Interface)

  • Tecnología moderna (estándar desde 2010 aproximadamente)
  • Interfaz gráfica con soporte para ratón
  • Usa tabla de particiones GPT (GUID Partition Table)
  • Soporta discos de cualquier tamaño (más de 2 TB sin problema)
  • Hasta 128 particiones por disco
  • Arranque mucho más rápido (pocos segundos o menos)
  • Secure Boot: verifica que el bootloader no ha sido modificado por malware

El bootloader: quién carga el sistema operativo

El bootloader es un pequeño programa que se ejecuta al inicio y cuya misión es localizar el kernel del SO en el disco y cargarlo en la RAM. Si tienes varios SO instalados, el bootloader te muestra un menú para elegir cuál quieres arrancar.

  • Windows Boot Manager: el bootloader de Windows. Normalmente no muestra menú a menos que detecte otro SO. Se configura con la herramienta bcdedit desde PowerShell o cmd.
  • GRUB (GRand Unified Bootloader): el más usado en Linux. Muestra un menú de inicio donde puedes elegir entre diferentes kernels o sistemas operativos instalados. Muy configurable y potente.
🔑 Concepto clave

Sin un bootloader funcionando correctamente, el sistema operativo no puede arrancar. Un error común es dañar el bootloader al instalar un segundo SO. Por eso existen herramientas de reparación de arranque tanto en Windows (Entorno de recuperación) como en Linux (Boot-Repair).

Instalación de Windows 11 en VirtualBox: requisitos, pasos y configuración inicial

⏱️ 110 min
RA1 CE-a · Parámetros básicos de instalación (10%) RA1 CE-c · Usar interfaz para preparar entorno (5%)

Para practicar sin riesgo en los ordenadores del aula, usaremos Oracle VirtualBox, un software de virtualización gratuito y de código abierto que permite crear ordenadores virtuales dentro de un ordenador físico real. Una máquina virtual (VM) es un ordenador simulado por software: tiene su propio disco duro virtual, su RAM virtual, su tarjeta de red virtual y puede ejecutar un sistema operativo diferente al de la máquina anfitriona sin interferir con él.

📌 ¿Cómo funciona la virtualización?
VirtualBox crea una capa de software que "engaña" al sistema operativo invitado haciéndole creer que dispone de hardware real. Cuando el SO invitado escribe en su "disco", en realidad está escribiendo dentro de un archivo grande almacenado en el disco de la máquina física. La CPU real ejecuta las instrucciones del SO invitado como si fueran las de cualquier otro programa. Esto permite ejecutar Windows dentro de Linux, Linux dentro de Windows, o varias versiones del mismo SO simultáneamente sin que interfieran entre sí.

El sistema de archivos: rutas, extensiones, tipos de archivos y estructura de directorios

⏱️ 110 min
RA1 CE-e · Manejo de archivos y periféricos (10%) RA1 CE-h · Conceptos básicos SO (15%)

El sistema de archivos es la forma en que el sistema operativo organiza toda la información en los discos. Entender su estructura es fundamental para encontrar, organizar y proteger los datos correctamente. Como técnico informático, pasarás gran parte de tu tiempo gestionando archivos: saber cómo se organizan es una habilidad esencial.

📌 Conceptos fundamentales del sistema de archivos

Archivo: conjunto de datos almacenados bajo un nombre único. Puede contener texto, imágenes, programas ejecutables, vídeos, etc. Todo lo que guardas en el ordenador es, en última instancia, un archivo.

Directorio (carpeta): contenedor lógico que agrupa archivos y otros directorios. Permite organizar la información de forma jerárquica, como las ramas de un árbol.

Ruta (path): la dirección exacta de un archivo dentro del sistema. Indica todos los directorios que hay que atravesar para llegar a él. Puede ser absoluta (desde la raíz del sistema) o relativa (desde la posición actual).

Extensión: las letras que aparecen después del último punto en el nombre del archivo (.docx, .pdf, .jpg). Indican al sistema operativo qué tipo de datos contiene y, normalmente, con qué programa debe abrirlo por defecto.

🌳 Diferencias en la estructura de directorios entre Windows y Linux:

🪟 Windows: basado en letras de unidad

Cada disco o partición recibe una letra: C: para el disco principal, D: para un segundo disco, E: para un DVD... Las rutas usan la barra invertida \ como separador. Ejemplo: C:\Users\alumno\Documentos\apuntes.txt

Carpetas principales:

  • C:\Windows\ → archivos del SO (¡no modificar ni borrar!)
  • C:\Program Files\ → programas instalados (64 bits)
  • C:\Users\[usuario]\ → carpeta personal de cada usuario
  • C:\Users\[usuario]\Documents\ → documentos
  • C:\Users\[usuario]\Downloads\ → descargas

🐧 Linux: árbol único con raíz /

En Linux no hay letras de unidad. Todo cuelga de una única raíz representada por /. Los discos y particiones adicionales se "montan" en directorios específicos. Las rutas usan la barra normal / como separador. Ejemplo: /home/alumno/Documentos/apuntes.odt

Directorios principales:

  • /home/[usuario]/ → tu carpeta personal
  • /home/[usuario]/Documentos/ → documentos
  • /home/[usuario]/Descargas/ → descargas
  • /etc/ → archivos de configuración del sistema
  • /usr/ → programas instalados

B10

Atributos de archivos: solo lectura, oculto, sistema, archivo y cómo gestionarlos

⏱️ 110 min
RA1 CE-d · Atributos y permisos en archivos/directorios (20%)

Cada archivo y carpeta almacena, además de su contenido, una serie de atributos (metadatos) que controlan ciertos comportamientos. En Windows, estos atributos son gestionados por el sistema de archivos NTFS. Comprenderlos es esencial para proteger información importante y para diagnosticar por qué un archivo "no se ve" o "no se puede modificar".

🔖 Los cuatro atributos principales en Windows (NTFS):

📖 Solo lectura (R - Read-only)

El archivo se puede abrir y leer, pero no se puede modificar ni eliminar (a menos que se quite el atributo). Si intentas guardar cambios, Windows te obliga a usar "Guardar como..." con otro nombre. Ideal para documentos finalizados o archivos de configuración que no deben alterarse.

🙈 Oculto (H - Hidden)

El archivo o carpeta no aparece en el Explorador de archivos a menos que tengas activada la opción "Mostrar archivos ocultos". No es una medida de seguridad real (cualquiera puede activar esa opción), pero evita modificaciones accidentales de archivos del sistema por parte de usuarios inexpertos.

⚙️ Sistema (S - System)

Marca archivos que son críticos para el funcionamiento del SO. Suelen estar también ocultos y ser de solo lectura. El sistema operativo los protege especialmente. Nunca modifiques ni elimines un archivo con atributo de sistema.

📦 Archivo (A - Archive)

Indica que el archivo ha sido modificado desde la última copia de seguridad. Los programas de backup usan este atributo para saber qué archivos necesitan ser copiados en la siguiente copia incremental. Al hacer la copia, el atributo se desactiva; al modificar el archivo, Windows lo activa automáticamente.

🔧 Cómo ver y cambiar atributos:

  • Windows (GUI): Botón derecho sobre el archivo → Propiedades → pestaña General. Marca/desmarca "Solo lectura" y "Oculto".
  • Windows (terminal): El comando attrib muestra y modifica atributos. attrib +r archivo.txt activa solo lectura. attrib +h +r doc.txt lo oculta y lo hace de solo lectura. attrib -h doc.txt le quita el atributo de oculto.
  • Linux: En Linux no existen estos atributos como tal. El equivalente a "oculto" es que el nombre del archivo empiece por un punto (.oculto.txt). Los permisos de Linux (que veremos en el B11) son el mecanismo equivalente pero mucho más potente.
🔑 Concepto clave

Los atributos de archivo no son medidas de seguridad. Cualquier usuario con acceso al archivo puede cambiarlos (excepto el atributo de sistema, que requiere permisos de administrador). Para una protección real, necesitas permisos NTFS (siguiente bloque).

B11

Permisos en archivos y carpetas: Windows (NTFS) vs Linux (chmod, chown)

⏱️ 110 min
RA1 CE-d · Atributos y permisos en archivos/directorios (20%)

Más allá de los atributos simples, los sistemas de archivos modernos implementan permisos de seguridad que controlan quién puede acceder a cada archivo y exactamente qué puede hacer con él. Este es uno de los pilares fundamentales de la seguridad informática en cualquier sistema operativo.

📌 Diferencia crucial entre atributos y permisos

Los atributos (solo lectura, oculto) son propiedades simples que afectan a todos los usuarios por igual. Los permisos son reglas de seguridad que especifican qué usuarios o grupos concretos pueden leer, escribir o ejecutar cada archivo. Los permisos son mucho más granulares y constituyen la verdadera base de la seguridad del sistema de archivos.

🔐 Permisos NTFS en Windows:

En Windows, los permisos se configuran desde las Propiedades del archivo → pestaña Seguridad. Cada archivo o carpeta tiene una Lista de Control de Acceso (ACL) donde se especifican los permisos para cada usuario o grupo. Los permisos básicos son:

  • Control total: permite leer, escribir, modificar, eliminar Y cambiar los permisos del archivo.
  • Modificar: permite leer, escribir, modificar y eliminar el archivo, pero NO cambiar sus permisos.
  • Leer y ejecutar: permite ver el contenido del archivo y, si es un programa, ejecutarlo.
  • Leer: permite ver el contenido pero no modificarlo ni eliminarlo.
  • Escribir: permite crear archivos nuevos dentro de una carpeta y modificar archivos existentes.

Los permisos NTFS se heredan automáticamente: una carpeta transmite sus permisos a todos los archivos y subcarpetas que contiene, a menos que se rompa explícitamente la herencia.

🐧 Permisos en Linux (sistema octal):

En Linux, cada archivo pertenece a un usuario propietario y a un grupo. Los permisos se dividen en tres categorías (dueño, grupo, otros) y tres acciones (lectura r=4, escritura w=2, ejecución x=1):

Lectura (r=4)Escritura (w=2)Ejecución (x=1)
Dueño (u)Puede leer el contenidoPuede modificarPuede ejecutar (si es programa)
Grupo (g)Los miembros pueden leerPueden modificarPueden ejecutar
Otros (o)Cualquier usuario puede leerPueden modificarPueden ejecutar

Ejemplos de permisos comunes en notación octal:

  • chmod 755 archivo → dueño: todo (rwx=7), grupo y otros: leer+ejecutar (5)
  • chmod 644 archivo → dueño: leer+escribir (6), grupo y otros: solo leer (4)
  • chmod 600 archivo → solo el dueño puede leer y escribir. Muy restrictivo.
⚠️ ¡Nunca uses chmod 777!

chmod 777 archivo da TODOS los permisos a TODO el mundo. Cualquier usuario del sistema podría leer, modificar e incluso borrar ese archivo. Es como dejar la puerta de tu casa abierta con un cartel que diga "pasen, cojan lo que quieran". Nunca uses 777 en archivos importantes, de configuración o del sistema.

🔑 Concepto clave

El principio de mínimo privilegio establece que cada usuario debe tener solo los permisos estrictamente necesarios para realizar su trabajo, y nada más. Si un usuario solo necesita leer un archivo, no le des permiso de escritura. Esto limita el daño que puede causar un error o un malware.

Exploración de soportes de almacenamiento: USB, discos duros, propiedades, formato

⏱️ 110 min
RA1 CE-f · Herramientas del SO para explorar soportes (10%) RA1 CE-d · Reconocer atributos y permisos (20%)

Los soportes de almacenamiento son los dispositivos físicos donde se guardan los datos de forma permanente. Como técnico informático, debes saber identificar sus características, comprobar su estado de salud, formatearlos con el sistema de archivos adecuado y expulsarlos de forma segura para no perder datos.

💾 Tipos principales de soportes de almacenamiento:

HDD (Disco duro mecánico)

Platos magnéticos giratorios + brazo lector. Lentos pero baratos y de gran capacidad (hasta 20 TB). Ideales para almacenamiento masivo y backups.

SSD (Unidad de estado sólido)

Memoria flash sin partes móviles. Mucho más rápidos que HDD (arranque del SO en segundos). Más caros por GB pero imprescindibles para el sistema operativo y programas.

USB / Pendrive / Tarjetas SD

Memoria flash portátil. Capacidades desde 4 GB hasta 1 TB. Conectan por USB o ranura SD. Ideales para transferir archivos entre equipos.

B13

Periféricos: instalación, drivers, administrador de dispositivos e impresoras

⏱️ 110 min
RA1 CE-e · Manejo de archivos y periféricos (10%)

Los periféricos son todos los dispositivos externos que conectamos al ordenador. Cada uno necesita un driver (controlador) para funcionar correctamente. Como técnico, gran parte de tu trabajo consistirá en instalar, actualizar y solucionar problemas de periféricos.

📌 ¿Qué es exactamente un driver?
Un driver es un pequeño programa (a veces solo unos pocos kilobytes) que actúa como traductor entre el sistema operativo y un dispositivo hardware concreto. El SO habla un "idioma" genérico; el driver traduce ese idioma a los comandos específicos que entiende el dispositivo. Sin el driver correcto, el dispositivo no funciona o funciona de forma limitada.

🖨️ Clasificación de periféricos:

Entrada

Envían información hacia el ordenador. Teclado, ratón, escáner, micrófono, webcam, lector de huellas dactilares.

Salida

Reciben información del ordenador. Monitor, impresora, altavoces, auriculares, proyector.

Entrada/Salida (E/S)

Ambas funciones. Pantalla táctil, disco duro externo, pendrive, tarjeta de red.

🔧 El Administrador de dispositivos en Windows:

Accesible desde Panel de control → Sistema → Administrador de dispositivos, o ejecutando devmgmt.msc. Muestra todos los dispositivos detectados, organizados por categorías. Los iconos más importantes que encontrarás:

  • ⚠️ Triángulo amarillo con exclamación: el dispositivo tiene un problema (driver incorrecto, conflicto con otro dispositivo, no funciona).
  • ❌ Cruz roja o flecha hacia abajo: el dispositivo está deshabilitado intencionadamente.
  • 🔵 Icono normal (sin advertencias): el dispositivo funciona correctamente.

🐧 Herramientas equivalentes en Ubuntu:

  • lsusb → lista todos los dispositivos conectados por USB.
  • lspci → lista los dispositivos conectados al bus PCI (tarjeta gráfica, red, sonido).
  • lshw → información detallada de TODO el hardware.
  • sudo dmesg → mensajes del kernel al detectar dispositivos (muy útil para depurar problemas).
🔑 Concepto clave

El 90% de los problemas con periféricos se resuelven con una de estas tres acciones: (1) actualizar el driver desde la web del fabricante, (2) desconectar y volver a conectar el dispositivo, o (3) reiniciar el equipo. Aprende este orden de actuación: driver → reconexión → reinicio.

B14

Antivirus y protección contra malware: tipos de amenazas, Windows Defender, ClamAV

⏱️ 110 min
RA1 CE-g · Operaciones de protección: antivirus, copias de seguridad (25%)

El malware (software malicioso) es cualquier programa diseñado para dañar, infiltrarse, espiar o tomar el control de un sistema informático sin el consentimiento de su propietario. La protección antivirus es la primera línea de defensa, pero no la única.

🦠 Tipos de malware más importantes:

Virus

Se adjunta a archivos legítimos y se propaga cuando alguien abre el archivo infectado. Necesita un "huésped" para existir. Como un virus biológico, depende de que el usuario lo ejecute.

Gusano (Worm)

Se copia a sí mismo automáticamente a través de la red. No necesita interacción humana ni adjuntarse a archivos. Explota vulnerabilidades del sistema operativo o de los programas instalados.

Troyano

Se disfraza de programa legítimo (un juego, una herramienta gratuita, un PDF falso) para engañar al usuario. Una vez ejecutado, abre una "puerta trasera" que permite al atacante controlar el equipo remotamente.

Ransomware

Cifra todos tus archivos personales (documentos, fotos, vídeos) y exige un rescate económico para desbloquearlos. Es la amenaza más grave actualmente. La mejor defensa: copias de seguridad (bloque B15).

Spyware

Espía tu actividad sin que lo sepas: registra las teclas que pulsas (keylogger), hace capturas de pantalla, roba contraseñas guardadas en el navegador. Suele venir con programas "gratuitos" de fuentes no confiables.

Adware

Muestra publicidad invasiva y no deseada. Aunque menos peligroso, consume recursos del sistema, ralentiza el equipo y a menudo rastrea tus hábitos de navegación para vender tus datos.

🛡️ Windows Defender (Microsoft Defender Antivirus):

Viene integrado gratuitamente en Windows 10 y 11. Es un antivirus completo que ha mejorado enormemente en los últimos años hasta el punto de que muchos expertos consideran que ya no es necesario instalar antivirus de terceros para uso doméstico. Funciones principales:

  • Protección en tiempo real: analiza cada archivo que abres, descargas o ejecutas antes de que pueda causar daño.
  • Firewall integrado: controla las conexiones de red entrantes y salientes.
  • Protección contra ransomware: "Acceso controlado a carpetas" impide que aplicaciones desconocidas modifiquen tus documentos sin permiso.

🐧 ClamAV en Linux:

Linux es mucho menos atacado que Windows, pero no es invulnerable. ClamAV es el antivirus de código abierto más usado. Se instala con sudo apt install clamav y se actualiza con sudo freshclam. Para escanear: clamscan -r /ruta.

🔑 Concepto clave

El mejor antivirus eres tú. La mayoría de infecciones ocurren porque el usuario (1) descargó un archivo de un sitio sospechoso, (2) abrió un adjunto de un correo desconocido, o (3) hizo clic en un enlace engañoso. La formación y el sentido común son la primera barrera de protección.

B15

Copias de seguridad: tipos de backup, estrategia 3-2-1, herramientas en Windows y Linux

⏱️ 110 min
RA1 CE-g · Operaciones de protección: copias de seguridad (25%)

Una copia de seguridad (backup) es una réplica exacta de tus datos almacenada en un lugar diferente al original. Es la protección más importante contra la pérdida de información por fallos de hardware, borrados accidentales, ataques de ransomware, incendios, robos o cualquier otro desastre.

⚠️ La regla de oro del backup

Si un dato no existe en al menos dos lugares diferentes, no existe en absoluto. No importa cuánto cuides tu ordenador: los discos duros fallan (todos, es cuestión de tiempo), los virus atacan, los errores humanos ocurren. Sin copia de seguridad, la pérdida de datos importantes es solo cuestión de tiempo.

📊 Los tres tipos de copias de seguridad:

Completa (Full)

Copia absolutamente todos los archivos seleccionados cada vez que se ejecuta. Es la más segura pero la más lenta y la que más espacio consume. Se suele hacer semanal o mensualmente como "base".

Incremental

Solo copia los archivos que han cambiado desde la última copia (sea completa o incremental). Muy rápida y eficiente en espacio. Inconveniente: para restaurar necesitas la última copia completa + TODAS las incrementales posteriores en orden.

Diferencial

Copia los archivos que han cambiado desde la última copia completa. Ocupa más espacio que la incremental (va creciendo cada día). Para restaurar solo necesitas la última completa + la última diferencial.

🔐 La estrategia 3-2-1 (estándar de la industria):

📌 Regla 3-2-1
  • 3 copias de tus datos: el original + dos copias de seguridad independientes.
  • 2 tipos de soporte diferentes: por ejemplo, un disco duro externo USB + almacenamiento en la nube. Así, si un tipo de soporte falla, el otro sobrevive.
  • 1 copia fuera de casa (off-site): en la nube, en casa de un familiar o en otra oficina. Así, si hay un incendio, inundación o robo en el edificio, tus datos sobreviven en otra ubicación.

Ejemplo práctico: (1) tus archivos en el ordenador, (2) backup diario en un disco duro externo USB, (3) backup semanal en la nube (Google Drive, OneDrive, pCloud). Si falla el ordenador, recuperas del disco externo. Si hay un incendio, recuperas de la nube.

Una copia de seguridad que nunca has probado a restaurar no es una copia de seguridad: es una esperanza. ⏱️ 110 min

Esta prueba práctica final integra todos los conocimientos y habilidades que has adquirido a lo largo del RA1. El objetivo es demostrar que eres capaz de instalar, configurar, personalizar, proteger y documentar un sistema operativo monousuario completo, trabajando de forma autónoma.

📌 Escenario de la prueba

Trabajas como técnico informático en una pequeña empresa. Te han encargado preparar un ordenador para un nuevo empleado que empieza mañana. Tienes 110 minutos para dejar el equipo completamente listo. Dispones de VirtualBox con una máquina virtual vacía.

🧪 Tareas a realizar (cada tarea indica el tiempo máximo recomendado):

  1. INSTALACIÓN (20 min): Instala Windows 11 desde cero en una nueva VM. Crea una cuenta local llamada empleado. No vincules ninguna cuenta Microsoft.
  2. PERSONALIZACIÓN (15 min): Cambia el fondo de escritorio, activa el modo oscuro, ancla a la barra de tareas: Explorador, Edge y Bloc de notas. Organiza el menú Inicio.
  3. ESTRUCTURA DE CARPETAS (10 min): Crea estas carpetas: C:\Empresa\Documentos\, C:\Empresa\Plantillas\, C:\Empresa\Copias\. Dentro de Documentos, crea 3 archivos de texto con contenido de prueba usando solo el Bloc de notas.
  4. SEGURIDAD (15 min):
    • Pon atributo de SOLO LECTURA a los archivos de Plantillas\.
    • Haz que la carpeta Copias\ esté OCULTA.
    • Configura Windows Defender con protección en tiempo real activada.
  5. COPIAS DE SEGURIDAD (15 min): Añade un segundo disco virtual a la VM. Configura el Historial de archivos para que haga copia automática de C:\Empresa\ en ese disco. Ejecuta la primera copia manualmente.
  6. DOCUMENTACIÓN (25 min): Crea un documento (en Writer o Bloc de notas) que incluya: nombre del equipo, usuarios creados, estructura de carpetas, atributos aplicados, configuración de seguridad realizada y configuración de copias de seguridad. Guarda el documento como DOCUMENTACION.txt en el escritorio.
  7. EXTRA (opcional, sube nota): Instala Ubuntu en otra VM y repite los pasos 2 a 5 adaptados a Linux. Documenta las diferencias.

Un sistema operativo no es un fin en sí mismo: es una herramienta. Lo importante no es saberse los menús de memoria, sino entender para qué sirve cada opción y cómo aplicarla para resolver problemas reales. Si has entendido los conceptos de este RA1, podrás enfrentarte a cualquier sistema operativo, incluso a uno que nunca hayas visto antes.
