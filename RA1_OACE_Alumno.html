B1

¿Qué es un sistema operativo? ¡Es el jefe de tu ordenador!

⏱️ 110 min
RA1 CE-h · Conceptos básicos SO (15%)

Imagina que tu ordenador es una orquesta. Tiene músicos (el procesador, la memoria, el disco duro...) pero sin director sería un caos: cada uno tocaría a su ritmo y no sonaría bien. El sistema operativo (SO) es ese director. Se encarga de que todos los componentes trabajen juntos y de que tú puedas usar el ordenador sin volverte loco.

📌 En palabras sencillas
El sistema operativo es el programa más importante de tu ordenador. Se enciende nada más arrancar y se encarga de que todo funcione. Sin él, no podrías abrir programas, guardar archivos ni navegar por Internet. Es como el director de orquesta que coordina todos los instrumentos.

🧩 Las 6 cosas principales que hace un SO por ti:

⚙️ 1. Reparte el trabajo del procesador

Cuando tienes varias aplicaciones abiertas (música, navegador, Word), el SO reparte el tiempo del procesador entre todas para que funcionen a la vez. ¡Como un malabarista!

🧠 2. Gestiona la memoria RAM

Decide qué datos guardar en la RAM (rápida) y cuáles en el disco (lento). Así los programas que más usas van más rápido.

💾 3. Organiza tus archivos

Crea carpetas, guarda documentos, fotos, vídeos... y sabe exactamente dónde está cada cosa en el disco duro.

🖨️ 4. Controla los periféricos

Cuando conectas un USB, una impresora o unos auriculares, el SO los detecta y los hace funcionar.

🔐 5. Protege tu información

Crea cuentas de usuario con contraseña. Así tus archivos están a salvo de miradas curiosas.

🖥️ 6. Te muestra todo en pantalla

El escritorio, las ventanas, los iconos... todo lo que ves y tocas con el ratón es la "cara visible" del SO.

🤔 ¿Y esto para qué me sirve?

Cuando entiendes qué hace un SO, puedes diagnosticar problemas tú mismo. ¿El ordenador va lento? Quizá tienes muchos programas abiertos y la RAM está llena. ¿No reconoce un USB? Puede que falte un driver. Saber esto te convierte en la persona que arregla los ordenadores de tu familia y amigos.

🔑 Idea clave para recordar

Sin sistema operativo, el ordenador es solo un montón de chips y cables que no hacen nada. El SO es lo que le da vida.

B2

Tipos de SO: ¿en qué se diferencian Windows, Linux, Android...?

⏱️ 110 min
RA1 CE-h · Conceptos básicos SO (15%)

No todos los sistemas operativos son iguales. Igual que hay coches deportivos, todoterrenos y furgonetas (cada uno para una cosa), hay SO para ordenadores, para móviles, para aviones... Vamos a poner orden en este lío.

👤 ¿Uno o varios usuarios?

Monousuario: solo lo usa una persona (ej: MS-DOS antiguo).
Multiusuario: varias personas, cada una con su cuenta (Windows, Linux).

⏱️ ¿Una cosa o varias a la vez?

Monotarea: un solo programa (MS-DOS).
Multitarea: música + navegador + Word a la vez (todos los modernos).

🔢 ¿32 o 64 bits?

Los de 64 bits pueden usar mucha más RAM y son más rápidos. Hoy casi todos son de 64 bits.

🔓 ¿Libre o de pago?

Libre: puedes ver y modificar el código (Linux, Ubuntu).
Propietario: el código es secreto (Windows, macOS).

🔑 Idea clave

"Software libre" no significa "gratis". Significa que tienes libertad para usarlo, modificarlo y compartirlo. Es como una receta de cocina que puedes cambiar a tu gusto.

🤔 ¿Y esto para qué me sirve?

Cuando instales programas o elijas un SO para un trabajo, sabrás qué necesitas. ¿Un SO ligero para un ordenador viejo? Linux. ¿El más compatible con todos los programas? Windows. ¿Para un móvil? Android o iOS.

B3

Las tripas del SO: kernel, shell, drivers... ¡no te asustes!

⏱️ 110 min
RA1 CE-h · Conceptos básicos SO (15%)

Un sistema operativo no es una sola pieza, sino varias capas que trabajan juntas. Como una lasaña: cada capa hace su trabajo y se comunica con la de al lado.

📌 Las 4 capas (de más profunda a más visible)

Imagínalo como una lasaña: cada capa solo habla con la que tiene al lado, nunca salta capas.

CapaNombre¿Qué hace?Ejemplo
🔩 1 (abajo)KernelHabla directamente con el hardware. Es el único que puede tocarlo.Kernel NT (Windows) · Kernel Linux
🔌 2DriversTraducen las órdenes del kernel para cada dispositivo concreto.Driver de tu impresora HP o ratón Logitech
🖥️ 3ShellLa "cara" del SO. Puede ser gráfica (GUI) o de texto (CLI).Escritorio de Windows · Terminal de Ubuntu
📱 4 (arriba)AplicacionesLos programas que usa el alumno.LibreOffice · Firefox · juegos
🔑 Idea clave

El kernel es el único que realmente toca el hardware. Todo lo demás le pide permiso al kernel. Es como el portero de una discoteca: nadie entra sin que él lo autorice.

🤔 ¿Y esto para qué me sirve?

Cuando algo falla, sabes dónde mirar. ¿No se oye la música? Driver de sonido. ¿Programa que no arranca? Kernel o permisos. ¿Pantalla congelada? Shell gráfico. Diagnosticar es saber buscar en la capa correcta.

B4

¿Qué pasa desde que pulsas el botón hasta que sale el escritorio?

⏱️ 110 min
RA1 CE-h · Conceptos básicos SO (15%)

Pulsas el botón de encendido y... ¡magia! En unos segundos aparece tu escritorio. Pero detrás de esa "magia" hay 10 pasos muy rápidos que ocurren siempre en el mismo orden. Conocerlos te ayudará a arreglar ordenadores que "no arrancan".

📌 La secuencia de arranque (versión resumida)
  1. ⚡ Llega la corriente
  2. 🔧 Se despierta la BIOS/UEFI (un chip en la placa base)
  3. 🩺 El ordenador se hace un "chequeo" rápido (POST): ¿funciona la RAM? ¿Está el teclado?
  4. 🔍 Busca discos duros, USB, DVD...
  5. 🎯 Elige desde dónde arrancar (disco duro, USB...)
  6. 📖 Lee el "cargador de arranque" (bootloader)
  7. 🧠 El bootloader carga el kernel del SO en la RAM
  8. 🚀 El SO despega y carga los drivers
  9. 🔔 Se inician los servicios (red, sonido...)
  10. 🔑 Aparece la pantalla para que escribas tu contraseña
🔑 Idea clave

Si el bootloader se daña, el ordenador no arranca aunque el disco esté perfecto. Es como perder la llave de tu coche: el coche está bien, pero no puedes arrancarlo.

🤔 ¿Y esto para qué me sirve?

Cuando un ordenador no arranca, puedes seguir estos 10 pasos para encontrar dónde se ha quedado atascado. ¿Se enciende pero no sale nada en pantalla? Fallo en el POST. ¿Sale el logo de Windows pero se reinicia? Fallo en el kernel o drivers.

B5

Instalar Windows 11 en VirtualBox: ¡tu primer ordenador virtual!

⏱️ 110 min
RA1 CE-a · Parámetros básicos de instalación (10%)

Ha llegado el momento de mancharse las manos. Vas a crear un ordenador virtual (un "ordenador de mentira" que funciona de verdad) e instalar Windows 11 en él. Lo mejor: si metes la pata, lo borras y empiezas de nuevo. ¡Sin miedo!

📌 ¿Qué es VirtualBox?
Es un programa gratuito que te permite crear ordenadores falsos dentro de tu ordenador real. Cada ordenador falso tiene su propio disco, RAM y pantalla. Puedes instalar Windows, Linux o lo que quieras sin tocar tu sistema real.

🖥️ Lo que necesita tu máquina virtual:

IngredienteCantidad mínima
RAM4 GB (4096 MB)
Procesadores2 núcleos
Disco duro virtual60 GB (formato VDI, dinámico)
ISO de Windows 11Descargada de Microsoft
⚠️ OJO: Virtualización activada

Para que VirtualBox funcione bien, la virtualización debe estar ACTIVADA en la BIOS. Lo comprobaste en el Bloque 4. Si no lo está, pide ayuda a tu profesor.

B8

Archivos, carpetas, rutas y extensiones: organiza tu vida digital

⏱️ 110 min
RA1 CE-e · Manejo de archivos (10%)

¿Alguna vez has buscado un archivo y no lo encontrabas? A todos nos ha pasado. La solución es organizar bien las carpetas y entender cómo funcionan las rutas y las extensiones. Es más fácil de lo que parece.

📌 Palabras que debes conocer

Archivo: cualquier cosa que guardas (un documento, una foto, una canción).
Carpeta (directorio): donde guardas los archivos, como un cajón.
Ruta: la dirección exacta para llegar a un archivo. Como "casa → cocina → cajón → tijeras".
Extensión: las letras después del punto (.docx, .pdf, .jpg) que dicen qué tipo de archivo es.

🪟 Windows

Ruta típica: C:\Users\TuNombre\Documentos\tarea.odt
Separador: \ (barra invertida)

🐧 Linux

Ruta típica: /home/tunombre/Documentos/tarea.odt
Separador: / (barra normal)

🔑 Idea clave

En Windows, si ocultas las extensiones, un archivo llamado factura.pdf.exe se verá como factura.pdf. Parece un PDF, pero es un virus. Activa siempre "Mostrar extensiones de archivo" en el Explorador.

B9

Crear, copiar, mover y borrar archivos como un profesional

⏱️ 110 min
RA1 CE-e · Manejo de archivos (10%)RA1 CE-f · Herramientas del SO (10%)

Crear, copiar, mover, renombrar y borrar archivos son las 5 operaciones que más vas a hacer en tu vida como técnico. Vamos a practicarlas de dos formas: con el ratón (GUI) y escribiendo comandos (terminal). La terminal parece cosa de hackers, pero cuando la dominas, es muchísimo más rápida que el ratón.

📁 Las 5 operaciones básicas (en GUI y en terminal):

OperaciónCon ratón (GUI)Windows (cmd)Ubuntu (bash)
📂 Crear carpetaBotón derecho → Nuevo → Carpetamkdir nombremkdir nombre
📄 Crear archivoBotón derecho → Nuevo → Documentoecho texto > archivo.txttouch archivo.txt
📋 CopiarCtrl+C, Ctrl+Vcopy origen destinocp origen destino
✂️ MoverCtrl+X, Ctrl+Vmove origen destinomv origen destino
🗑️ BorrarSupr (papelera)del archivo (papelera)rm archivo (⚠️ permanente)
⚠️ ¡Cuidado con rm en Linux!

En Linux, rm borra los archivos para siempre. No hay papelera, no hay "deshacer". Antes de usar rm, piensa dos veces. Si quieres ir con cuidado, usa rm -i (te preguntará antes de borrar cada archivo).

🤔 ¿Y esto para qué me sirve?

Imagina que tienes que organizar 500 fotos en carpetas por fecha. Con el ratón tardarías horas. Con la terminal, escribes 3 comandos y listo. La terminal es tu superpoder secreto como técnico.

🔑 Idea clave

Los comandos mkdir, cp y mv funcionan prácticamente igual en Windows y Linux. Apréndelos una vez y te servirán para los dos sistemas.

B10

Atributos de archivos: solo lectura, oculto... ¿cómo proteger tus cosas?

⏱️ 110 min
RA1 CE-d · Atributos y permisos (20%)

Cada archivo tiene "etiquetas invisibles" llamadas atributos que le dicen al sistema cómo comportarse. Por ejemplo: "este archivo no se puede modificar" (solo lectura) o "este archivo no se ve" (oculto). Vamos a jugar con ellos.

📌 Los 4 atributos principales en Windows

📖 Solo lectura (R): se puede leer, pero NO modificar ni borrar (a menos que quites el atributo).
🙈 Oculto (H): el archivo desaparece del explorador. No es magia: solo está "escondido".
⚙️ Sistema (S): archivos importantes para Windows. ¡No los toques!
📦 Archivo (A): avisa a las copias de seguridad de que este archivo ha cambiado.

🔧 Cómo ver y cambiar atributos desde la terminal (cmd):

attrib archivo.txt              # Ver atributos
attrib +r archivo.txt            # Poner solo lectura
attrib +h archivo.txt            # Ocultar archivo
attrib -r -h archivo.txt         # Quitar solo lectura y oculto
🤔 ¿Y esto para qué me sirve?

¿Quieres que nadie modifique tu trabajo final? Solo lectura. ¿Quieres esconder un archivo de miradas curiosas? Oculto. Pero recuerda: los atributos no son contraseñas, cualquiera que sepa puede quitarlos.

🔑 Idea clave

En Linux no existen estos atributos. En su lugar, los archivos que empiezan por punto (.secreto) están ocultos. Y los permisos (siguiente bloque) hacen el trabajo de proteger los archivos de verdad.

B11

Permisos: quién puede tocar tus archivos y quién no

⏱️ 110 min
RA1 CE-d · Atributos y permisos (20%)

Los permisos son como las llaves de las habitaciones de tu casa. Tú (el administrador) tienes la llave maestra. Tu hermano (usuario estándar) solo puede entrar en su habitación. Un invitado no puede entrar en ninguna. Así funcionan los permisos en un sistema operativo.

📌 Permisos en Windows (NTFS)

Se configuran en Propiedades → Seguridad. Los 5 niveles, de mayor a menor poder:

Nivel¿Qué puede hacer?Típico para...
🔑 Control totalTodo: leer, escribir, borrar y cambiar permisosAdministrador
✏️ ModificarLeer, escribir y borrar (no cambiar permisos)Usuario de confianza
▶️ Leer y ejecutarVer contenido y ejecutar programasUsuario normal
👁️ LeerSolo ver el contenido, no modificar nadaConsultor externo
📝 EscribirCrear y modificar archivos, pero no borrarCaso muy específico

🐧 Permisos en Linux (chmod):

En Linux, los permisos se representan con números (notación octal). Cada dígito es la suma de: Lectura=4, Escritura=2, Ejecución=1.

ComandoSignificado¿Quién puede qué?
chmod 755rwx r-x r-xDueño: todo. Grupo y otros: leer+ejecutar
chmod 644rw- r-- r--Dueño: leer+escribir. Grupo y otros: solo leer
chmod 600rw- --- ---Solo el dueño puede leer y escribir
⚠️ ¡Nunca uses chmod 777!

chmod 777 da TODOS los permisos a TODO el mundo. Es como dejar la puerta de tu casa abierta con un cartel de "pasen y cojan lo que quieran".

🔑 Idea clave

El principio del mínimo privilegio dice: cada persona debe tener solo los permisos que necesita para trabajar, y nada más. Si alguien solo necesita leer, no le des permiso de escribir.

B12

Discos duros, SSD, USB: todo sobre dónde guardas tus cosas

⏱️ 110 min
RA1 CE-f · Herramientas del SO (10%)RA1 CE-d · Atributos y permisos (20%)

Tus fotos, tus trabajos, tus juegos... todo se guarda en dispositivos de almacenamiento. Vamos a conocer los tipos que hay, cómo se formatean y qué sistema de archivos usa cada uno. Porque no es lo mismo un USB que un disco duro de 4 TB.

💾 Tipos de almacenamiento:

🔄 HDD

Disco duro de toda la vida. Platos que giran. Barato, mucha capacidad, pero lento. Ideal para guardar muchas fotos o backups.

⚡ SSD

Sin partes móviles. Muy rápido. El ordenador arranca en segundos. Más caro. Ideal para el sistema operativo.

🔌 USB

Pequeño, portátil, se conecta al puerto USB. Perfecto para llevar archivos de un sitio a otro.

📊 Sistemas de archivos (cada disco tiene uno):

Sistema¿Dónde se usa?Tamaño máximo de archivo
NTFSWindowsIlimitado (en la práctica)
ext4Linux16 TB
FAT32USB, cámaras4 GB (⚠️ no caben archivos grandes)
exFATUSB modernosIlimitado (compatible con Windows, Mac y Linux)
🤔 ¿Y esto para qué me sirve?

¿Tu USB de 64 GB no te deja copiar una película de 5 GB? Está en FAT32. Tienes que formatearlo a exFAT o NTFS. Saber esto te ahorra muchos quebraderos de cabeza.

🔑 Idea clave

Siempre expulsa el USB antes de desconectarlo. Si lo arrancas sin avisar, puedes perder datos o estropear el sistema de archivos. En Windows: "Quitar hardware con seguridad". En Linux: botón "Expulsar".

B13

Impresoras, ratones, teclados y todo lo que conectas al ordenador

⏱️ 110 min
RA1 CE-e · Manejo de archivos y periféricos (10%)

Todo lo que conectas al ordenador (por USB, Bluetooth, HDMI...) se llama periférico. Cada uno necesita un driver (un pequeño programa) para funcionar. Vamos a poner orden en este mundo de cables y conexiones.

🖨️ Los 3 tipos de periféricos:

📥 Entrada

Mandan información al ordenador. Teclado, ratón, micrófono, webcam, escáner.

📤 Salida

Reciben información del ordenador. Monitor, impresora, altavoces, proyector.

🔄 Entrada/Salida

Hacen las dos cosas. Pantalla táctil, pendrive, disco duro externo.

🔧 El Administrador de dispositivos en Windows: ejecuta devmgmt.msc. Ahí ves todos los periféricos. Si alguno tiene un triángulo amarillo ⚠️, significa que tiene problemas (le falta el driver o está mal configurado).

🤔 ¿Y esto para qué me sirve?

El 90% de los problemas con periféricos se arreglan de 3 formas: (1) actualizar el driver desde la web del fabricante, (2) desconectar y volver a conectar, (3) reiniciar. Apréndete este orden: driver → reconexión → reinicio.

B14

Virus, troyanos y cómo protegerte: ¡que no te engañen!

⏱️ 110 min
RA1 CE-g · Operaciones de protección (25%)

Un malware es un programa malo que quiere dañar tu ordenador, robarte datos o espiarte. Hay muchos tipos, como los virus de la gripe: cada uno ataca de una forma. Conocerlos es el primer paso para protegerte.

🦠 Los 6 tipos de malware más comunes:

🦠 Virus

Se pega a archivos normales. Cuando abres el archivo, infecta el sistema.

🐛 Gusano

Se copia solo por la red. No necesita que hagas nada. Muy peligroso.

🐴 Troyano

Se disfraza de programa útil. Cuando lo instalas, abre una puerta trasera al atacante.

🔒 Ransomware

Secuestra tus archivos y pide dinero. La mejor defensa: copias de seguridad.

👁️ Spyware

Te espía sin que lo sepas. Registra lo que escribes, las webs que visitas...

📢 Adware

Te llena la pantalla de anuncios. Molesto pero menos peligroso.

🛡️ Windows Defender: viene gratis con Windows 10 y 11. Es un antivirus muy bueno. Asegúrate de que está activado (Seguridad de Windows → todos los checks en verde ✅).

🐧 ClamAV en Ubuntu:

sudo apt install clamav       # Instalar
sudo freshclam                 # Actualizar base de datos de virus
clamscan -r ~/Descargas        # Escanear carpeta
🔑 Idea clave

El mejor antivirus eres tú. La mayoría de infecciones ocurren porque alguien abrió un archivo sospechoso o hizo clic en un enlace trampa. Si algo parece demasiado bueno para ser verdad, es mentira.

B15

Copias de seguridad: no pierdas tus archivos nunca más

⏱️ 110 min
RA1 CE-g · Operaciones de protección (25%)

Imagina que mañana enciendes el ordenador y... ¡todos tus archivos han desaparecido! Fotos, trabajos, apuntes... todo perdido. Esto pasa de verdad más a menudo de lo que crees. La solución: copias de seguridad.

⚠️ La regla de oro

Si un archivo no existe en al menos 2 sitios diferentes, NO existe. Los discos duros fallan. Los virus atacan. Los errores humanos ocurren. Sin copia de seguridad, perder tus datos es cuestión de tiempo.

📊 Los 3 tipos de copias de seguridad:

📦 Completa

Copia TODO cada vez. La más segura, pero lenta y ocupa mucho.

➕ Incremental

Solo copia lo que ha cambiado desde la última copia. Rápida y ocupa poco.

📊 Diferencial

Copia lo que ha cambiado desde la última copia COMPLETA. Término medio.

🔐 La estrategia 3-2-1 (la que usan los profesionales):

  • 3 copias de tus datos (el original + 2 backups)
  • 2 tipos de soporte diferentes (disco externo + nube, por ejemplo)
  • 1 copia fuera de casa (en la nube o en otra ubicación, por si hay incendio o robo)
🔑 Idea clave

Una copia de seguridad que nunca has probado a restaurar no es una copia: es una esperanza. Cada cierto tiempo, intenta recuperar un archivo de tu backup para comprobar que funciona.
